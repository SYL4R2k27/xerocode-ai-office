# L01 · Лендинг — `/` (public marketing site)

> Глубокая дизайн-спецификация публичного сайта. Источник: `XEROCODE_FULL.html` строки 1786–1829 (hero L01) + бренд-секции PART I (8 воркспейсов/палитра/типографика, строки 1618–1784) + принципы (2808–2850). Остальные секции спроектированы из дизайн-системы (помечено «ПРЕДЛОЖЕНИЕ»).
> Токены — именами CSS-переменных. Иконки — lucide-react. Копирайт — русский, технический, без хайпа. Запрещёнка соблюдается строго (это самое заметное место бренда).

---

## 0. Назначение и принципы
**Задача лендинга:** за один экран объяснить, что XeroCode — это AI-офис на 8 воркспейсов для одного человека/малой команды, с BYOK и российским стеком, и довести до «Начать бесплатно». Тон — спокойный, инженерный, доказательный. **Никакого** «AI-powered/next-gen/революционно», улыбающихся стоков, роботов, неона, 3D-блобов.

**Поверхность:** Carbon (`--bg`) базовая; секции чередуются `--bg` / `--bg-soft` / редкие **Cream**-секции (`--cream`) для контраста ритма (как PART I «8 воркспейсов» на cream). Акцент — Violet `--xero`. Sienna `--sylar` — только как DNA-маркер связи с SYLAR/RoleFlow.

**Сетка/ритм:** `--pad-x clamp(20px,4vw,64px)`, вертикальный ритм секций `padding: 140px var(--pad-x)` (на мобиле 80px). Контент max-width ~1200px. Hairlines `1px var(--line-bg)`, теней почти нет.

**Подпись движения:** все reveal — `--ease cubic-bezier(0.22,1,0.36,1)`, появление `opacity 0→1 + translateY(40→0)` ~1s по IntersectionObserver. Hero — **karaoke scroll-reveal** (подсветка по словам). CTA — **magnetic** (лёгкое притягивание к курсору ≤6px). Всё через `motion/react`.

## 2. [00] Nav — sticky
**Раскладка:** `flex space-between`, `padding: 22px var(--pad-x)`, фон `--bg` с `backdrop` при скролле (без glass-блюра — просто плотный `--bg` + нижний hairline `--line-bg` появляется после 40px скролла).
- **Лого** слева: «XeroCode•» (`--display` 600, 18px, `-0.03em`) + точка-дот `--xero` 6px.
- **Меню** центр/право: `Возможности · Тарифы · Документы · Войти` (`--body` 12.5px, `--on-bg-mute`, hover → `--on-bg`, opacity-transition 200ms).
- **CTA** справа: pill «Начать бесплатно» (фон `--xero`, текст `--cream`, radius 999, 8×16, magnetic hover; суффикс-стрелка `↗` сдвигается на hover).
**Поведение:** скролл → nav уплотняется (padding 22→14px), появляется hairline. Активный раздел подсвечивается при скролле (scroll-spy). 
**Адаптив <768:** меню → бургер (`Menu` icon) → full-screen overlay-меню (`--bg`, крупные пункты `--display`); CTA остаётся.
**lucide:** `Menu`, `X`, `ArrowUpRight`.

## 3. [01] Hero — karaoke scroll-reveal
**Из мокапа:** eyebrow «▸ AI-офис для одного человека» (mono, `--xero`, +22% tracking). **H1** (`--display` 500, `clamp(40px,5vw,80px)`, line-height 0.95, `-0.04em`):
> Один человек. **Восемь воркспейсов.** Любая задача за минуты.
(«Восемь воркспейсов» — `--xero`). **Sub** (`--body` 17px, `--on-bg-mute`, max-width 560): «Чат, документы, картинки, видео, код, звук — в одном интерфейсе. 14 моделей. BYOK на всех тарифах. Российский стек: Yandex GPT, GigaChat, Pollinations.»
**CTA-пара:** pill primary «Начать бесплатно →» (`--xero`/`--cream`) + ghost «Посмотреть демо» (прозрач + border `--line-bg-strong`).
**Karaoke-механика:** слова H1 стартуют на `--on-bg-dim`, по мере скролла/входа подсвечиваются до `--on-bg` (ключевые — до `--xero`) словно караоке; реализация — span на слово, прогресс по scroll-позиции. ≤ длительности секции, плавно `--ease`. На `<768` — упрощить до фразовой подсветки (3 строки), не пословной (производительность).
**Состояния:** при reduce-motion — мгновенный полноцветный H1 без караоке.
**lucide:** `ArrowRight`.

## 4. [02] Product shot — browser-frame
**Назначение:** сразу показать продукт (не абстракции). Крупный browser-frame (3 точки + url `app.xerocode.ru`) с мокапом **дашборда или чата** (A02/A03). Лёгкий reveal снизу + параллакс ≤8px.
**Компоненты:** `browser-bar` (dots + url), `browser-body` со скриншотом. Рамка — `--bg-soft`, border `--line-bg`, radius 12.
**ПРЕДЛОЖЕНИЕ:** табы-переключатель под скрином («Чат · Картинки · Код · Оркестрация») — клик меняет скриншот, демонстрируя 8 воркспейсов одним блоком.
**Адаптив:** на мобиле — один скрин, свайп между воркспейсами.

## 6. [04] 8 воркспейсов — ws-grid (на Cream)
**Из PART I.** Секция `--cream` (контрастный светлый блок — приём ритма). 
- **Заголовок:** section-num «01 — Воркспейсы» (mono, `--on-cream-mute`), title «Восемь модулей. Один интерфейс.» (`--display` 500, 40px, `--on-cream`), sub про переключение в один клик + перенос контекста.
- **ws-grid** (`repeat(4,1fr)`, на 768 → 2 кол, <768 → 1): 8 карточек. Каждая: ws-icon (2-буквенный код CH/TX/IM/CD/VD/SD/CR/OR на плашке цвета воркспейса), ws-tag (mono «▸ 01 · CHAT»), ws-name, ws-desc. Цвета воркспейсов — **только в ico/tag** (правило), карточка остаётся cream.
- 8 воркспейсов: Чат(`--ws-chat`) / Текст(`--ws-text`) / Картинки(`--ws-images`) / Код(`--ws-code`) / Видео(`--ws-video`) / Звук(`--ws-sound`) / Корпоратив(`--ws-corp`, бейдж «Enterprise») / Оркестрация(`--ws-orchestr`).
**Интеракции:** hover карточки → border `--xero`/подъём 2px; клик → якорь на детальную под-секцию или demo. 
**lucide:** по воркспейсам — `MessageSquare, FileText, Image, Code2, Video, AudioLines, Building2, Workflow`.

## 7. [05] Ценность — «один делает работу пятерых» (3 шага)

## 17. Состояния / Адаптив / Производительность
- **Reduce-motion:** отключить караоке/параллакс/magnetic — оставить мгновенный contentful render.
- **Адаптив:** ≥1280 полные сетки; 768–1279 — 2-кол грид-секции, hero clamp уменьшает; <768 — всё в 1 колонку, nav→бургер, вертикальный ритм 80px, караоке→фразовое, product-shot свайп.
- **Перф:** изображения lazy, шрифты preconnect+swap (как в мокапе), анимации только transform/opacity, спиннеров нет.
- **SEO/мета** (ПРЕДЛОЖЕНИЕ): title/description с «AI-офис», OG-картинка browser-frame, h-иерархия, lang=ru.

# A04 · Картинки — `/images`

> Глубокая дизайн-спецификация одного экрана XeroCode v3 (Workspace 03 · Картинки).
> Источник правды по разметке: `XEROCODE_FULL.html` стр. 2111–2174. Источник правды по токенам: `XEROCODE_REDESIGN_BRIEF.md` §2 и `Enhance Office View Design/src/styles/xerocode-v3.css` (`.app-layout` стр. 532–647, `.img-*` стр. 1047–1120).
> Акцент воркспейса: **`--ws-images` `#EC4899` (pink)** — только в badge / иконке воркспейса / dot / точечных индикаторах прогресса. PRIMARY действий остаётся **`--xero` `#7C5CFF` (violet)**.
> Закон бренда соблюдён: без курсива/serif, без градиентных заливок на крупных площадях, без glassmorphism/glow/частиц, без спиннеров > 200 мс. (⚠️ Градиенты `.img-card.c1…c8` и `backdrop-filter: blur(8px)` в мокапе — это **скелетон-заглушки превью**, в продакшене заменяются растром генерации; см. раздел «Регионы» → img-grid и «Маппинг на код».)

---

## 1. Назначение

Рабочее пространство генерации изображений по текстовому промпту. Один экран закрывает весь цикл: ввод промпта → выбор стиля/размера/модели → генерация (batch до 4) → просмотр сетки результатов → действия над карточкой (скачать / вариации / в полный размер / удалить).

Ключевые задачи пользователя:
- быстро получить картинку под рабочую задачу (баннер, иллюстрация к посту, обложка, иконка, мокап);
- осознанно выбрать «бесплатно vs премиум» — провайдер виден и влияет на лимит;
- держать перед глазами расход месячного лимита генераций (фича тарифа, не прячем);
- вернуться к ранее сгенерированному (история в сетке).

Позиционирование внутри продукта: третий из 8 воркспейсов. Контекст переносится из других воркспейсов (промпт может прийти из Чата через хэндофф «Запускаю в воркспейсе Картинки»).

## 2. Доступ (тариф / лимит)

| Тариф | Доступ к `/images` | Месячный лимит генераций | Провайдеры |
|---|---|---|---|
| **Free** | ✅ есть | **30 изображений / мес** (счётчик `12 / 30`) | Pollinations (free) только |
| **Pro** | ✅ есть | расширенный (напр. 300/мес) | Pollinations + gpt-image-2 (премиум) |
| **Enterprise** | ✅ есть | по договору / pooled на организацию | все + приоритет очереди |
| **BYOK** (любой тариф) | ✅ есть | **∞** при своём ключе провайдера | свой ключ gpt-image-2 / Pollinations |

- Лимит `12 / 30` в тулбаре — это `использовано / квота` за календарный месяц (mono, `--on-bg-dim`). Цифра монотонно растёт до сброса 1-го числа.
- **Единица лимита = одна генерация-карточка.** Batch из 4 карточек = −4 от остатка (см. Edge cases → batch).
- **Pollinations не тратит лимит при BYOK**, но в кадре по умолчанию (на тарифе) тратит — провайдер free, но квота генераций тарифная.
- Премиум-модель `gpt-image-2` на Free **недоступна**: pill `▸ Model: gpt-image-2` показывается как locked (см. «Поведение» → выбор провайдера).
- Org-isolation: история и лимит привязаны к `organization_id` пользователя (правило проекта №7). Карточку нельзя открыть/удалить по чужому `image_id`.

## 4. Регионы и компоненты

### 4.1 `app-side` — сайдбар (220px, общий примитив)

- **Активный пункт «Картинки»** `li.active`: `background: --xero-mute` · текст `--xero` · иконка `--xero` opacity 1.

- **Счётчик `.num` = `12`** у пункта «Картинки»: mono 9.5px · `background: --xero` · текст `--cream` · radius 999px · padding 1/6. Это бейдж «использовано в этом месяце» (дублирует левую часть toolbar-счётчика).
- **lucide-иконки** пунктов: Чат `MessageSquare` · Текст `FileText` · **Картинки `Image`** · Код `Code` · Видео `Video` · Звук `AudioLines` · Оркестрация `Workflow` · База знаний `Library` · Документы `Files` · API-ключи `KeyRound` · Настройки `Settings`.

### 4.5 `img-grid` + `img-card` — сетка результатов

Источник: `.img-grid` (css 1085–1089), `.img-card` (1090–1120).

- **Grid**: `display:grid` · `grid-template-columns: repeat(4, 1fr)` · gap 10. 8 карточек = 2 ряда × 4 (адаптив 4→2→1).
- **Card `.img-card`**: `aspect-ratio: 1` (квадрат) · radius 10 · `position:relative` · `overflow:hidden` · `border: 1px solid --line-bg`.
- **Превью**:
  - В мокапе — `.placeholder` с номером «01…08» (Space Grotesk 500 · 28px · −0.025em · `--cream`) на цветном `linear-gradient` (`.c1…c8`). ⚠️ **Это скелетон.** В продакшене `.img-card` содержит `<img>` растра генерации (`object-fit: cover`), номер/градиент не показываются.
  - **Скелетон загрузки** (до прихода растра) сохраняет тот же квадрат: ровная заливка `--bg-tile` + по центру номер позиции `--on-bg-dim` (БЕЗ градиента в проде — градиент только в HTML-мокапе как декор) + индикатор прогресса (см. Состояния → loading).
- **Meta `.img-card .meta`**: `position:absolute` низ · padding 10/14 · `background: rgba(10,10,11,0.6)` + `backdrop-filter: blur(8px)` · `font-family: --mono` · 9.5px · letter-spacing .18em · UPPERCASE · `--cream`.
  - Содержимое (из мокапа): «Editorial · 1024px» / «Photo · 2K» / «Illustration» / «3D · Cinema» / «Photo · 4K» / «3D · iso». = `{Style} · {разрешение/режим}`.
  - ⚠️ `backdrop-filter: blur` формально близок к glassmorphism. **Решение:** это узкая нижняя плашка-подпись (не панель), читаемость поверх фото обязательна → blur допустим как утилитарный контраст-слой, БЕЗ свечения/прозрачных бордеров/неона. В проде допускается заменить на плотный `linear-gradient` низа `rgba(10,10,11,0)→rgba(10,10,11,.7)` (контрастная подложка подписи, не «стекло»).
- **Hover-действия карточки** (НЕ нарисованы в мокапе — проектируются): при hover на `.img-card` поверх превью проявляется тёмный оверлей `rgba(10,10,11,.45)` (180ms `--ease`) с рядом кнопок-иконок (40×40, `--cream`, hover-кружок `rgba(244,239,227,.12)`):

| Действие | lucide | Поведение |
|---|---|---|
| В полный размер | `Maximize2` / `Expand` | открыть lightbox-модал (см. Поведение) |
| Скачать | `Download` | скачать PNG/JPG/WebP оригинал |
| Вариации | `Layers` / `Copy` | сгенерировать 4 вариации этого изображения (img2img, −лимит) |
| Регенерировать | `RefreshCw` | повторить с тем же промптом+seed±1 |
| Копировать промпт | `ClipboardCopy` | скопировать промпт карточки в `img-prompt` |
| Удалить | `Trash2` | удалить из истории (подтверждение) |

  - Верхний-правый угол карточки: маленький **badge провайдера** (mono 9px) — `FREE` (Pollinations, `--on-bg-mute`) или `PREMIUM` (gpt-image-2, `--ws-images` точечно). Помогает читать «что тратило лимит/деньги».
  - Доступность: ряд действий достижим с клавиатуры (карточка `tabindex`, действия — настоящие `<button>`); на тач-устройствах оверлей по тапу (см. Адаптив).

## 5. Поведение и интеракции

### 5.1 Ввод промпта и генерация
1. Пользователь печатает в `img-prompt`. Пустой промпт → `gen-btn` disabled.
2. Submit: клик «Сгенерировать» **или** `Ctrl/Cmd + Enter` в поле. Просто `Enter` = перенос строки (промпт многострочный).
3. На submit формируется запрос `{prompt, style, size, model/provider, batch_n}` → создаются N карточек-плейсхолдеров в начале сетки (новые сверху), статус `loading`.
4. Лимит уменьшается оптимистично на `batch_n`; при ошибке генерации возврат к остатку.

### 5.2 Выбор Style / Size / Model
- **Style**: клик по пилюле → она `.active`, остальные в группе теряют active. Применяется к следующей генерации (не ретроактивно).
- **Size**: клик по `▸ Size:` → dropdown: `1024×1024` (1:1) · `1536×1024` (3:2) · `1024×1536` (2:3) · `512×512` (быстро/free). Выбор обновляет значение в пилюле.
- **Model/Provider**: клик по `▸ Model:` → dropdown:
  - `Pollinations` — FREE-бейдж, доступен всем, помечен `Star`-pin как дефолт на Free;
  - `gpt-image-2` — PREMIUM-бейдж; на Free locked (`Lock` + «обнови тариф»); на Pro+/BYOK доступен.
- Изменение Style/Size/Model **не запускает** генерацию — только настраивает следующий запуск (явный submit обязателен).

### 5.3 Действия над карточкой (hover/tap)
- **В полный размер** (`Maximize2`): открывает **lightbox-модал** по центру (затемнение фона `rgba(10,10,11,.7)`, БЕЗ blur-стекла): крупное превью + панель meta (промпт целиком, style/size/model, seed, провайдер, дата) + те же действия (Download/Вариации/Регенерировать/Удалить) + стрелки ‹ › для навигации по сетке. Закрытие: `Esc`/клик-фон/`X`.
- **Скачать** (`Download`): меню формата PNG / JPG / WebP; имя файла `xerocode_{style}_{date}_{seq}.png`.
- **Вариации** (`Layers`): генерирует 4 новых карточки на основе изображения (тот же промпт, новые seed) → тратит лимит как batch-4.
- **Регенерировать** (`RefreshCw`): новая генерация тем же промптом (новый seed) → 1 карточка, −1 лимит.
- **Удалить** (`Trash2`): инлайн-подтверждение в карточке («Удалить? Да / Отмена», mono, `--sienna` для «Да») — без отдельного модала. Удаление из истории (лимит НЕ возвращается — генерация уже потрачена).

### 5.4 Выбор провайдера free vs paid (как коммуницируем)
- Бейдж `FREE`/`PREMIUM` на каждой карточке + значение в пилюле `▸ Model`.
- На Free тарифе: дефолт Pollinations; gpt-image-2 locked с явным CTA «обнови тариф».
- На Pro+: переключение свободное; рядом с premium-моделью подсказка-tooltip «Тратит лимит быстрее / выше качество».
- BYOK: если подключён свой ключ (Настройки → API-ключи), бейдж `BYOK ∞`, лимит не тратится — текст счётчика меняется на «∞ · свой ключ».

## 6. Состояния

### 6.1 Пусто / первая генерация (0 изображений)
- `img-grid` скрыт, вместо него — центрированный empty-state (на `--bg`, в духе A12):
  - mono-eyebrow «▸ С ЧЕГО НАЧАТЬ» (`--xero`);
  - заголовок «Опиши картинку — нарисуем» (Space Grotesk 500 · ~28px · `--on-bg`);
  - sub «Например: баннер для поста, иконка приложения, иллюстрация к статье» (`--on-bg-mute`);
  - **3 стартовых шаблона-пилюли** (заполняют `img-prompt` при клике): «Баннер для соц. сетей» · «Иконка приложения» · «Обложка статьи» (lucide `ImagePlus` / `AppWindow` / `Newspaper`).
- Счётчик показывает `0 / 30`.

### 6.2 Loading генерации — БЕЗ спиннера > 200 мс
Закон бренда: спиннеры дольше 200 мс запрещены. Генерация картинки — операция 3–30 с, поэтому прогресс показываем **детерминированно, без крутилки**:
- Карточка-плейсхолдер: заливка `--bg-tile`, по центру **номер позиции** + статус-строка mono («Генерация…»/«В очереди…»).
- **Индикатор прогресса — тонкая `2px` линия по низу карточки** (под будущей meta), ширина растёт по реальному прогрессу провайдера; если провайдер не отдаёт прогресс — **shimmer-проход** (горизонтальный градиент-блик, бесконечный, как skeleton-loading — это НЕ спиннер; допустимо для скелетонов). Цвет линии `--ws-images`.
- Текстовый ETA (mono, `--on-bg-dim`): «~8 с» если провайдер даёт оценку.
- Появление готового растра: `opacity 0→1` + лёгкий `scale .98→1` за 200 мс `--ease` (без вспышек/частиц).
- batch: все N плейсхолдеров видны сразу, заполняются по мере готовности (могут приходить вразнобой).

### 6.3 Ошибка генерации
- Карточка переходит в error-вид: заливка `--bg-tile`, иконка `AlertTriangle` (`--sienna`), текст «Не удалось» (mono) + кнопка `RefreshCw` «Повторить».
- Лимит возвращается (+1 за упавшую карточку).
- Toast снизу (mono, `--bg-soft` + `--line-bg-strong`): «Провайдер недоступен — попробуйте другую модель» с действием «Сменить модель».

### 6.4 Лимит достигнут (30 / 30)
- Счётчик `30 / 30` → текст `--sienna` (error-токен), без мигания.
- `gen-btn` disabled; при наведении tooltip «Лимит месяца исчерпан».
- Под `img-prompt` появляется баннер-полоска (`--bg-soft` · `--line-bg-strong` · radius 12): «Лимит 30/30 исчерпан. Сбросится 1-го числа.» + CTA `pill` «Перейти на Pro» (`--xero`) и вторичная ссылка «Подключить свой ключ (BYOK)».
- Существующие карточки полностью интерактивны (просмотр/скачать/удалить работают; вариации/регенерация — заблокированы как новые генерации).

### 6.5 NSFW / отказ генерации
- Если провайдер/модерация отклонили промпт: карточка-плейсхолдер → reject-вид (иконка `ShieldAlert`, `--on-bg-mute`), текст «Запрос отклонён модерацией».
- Лимит **не тратится** (возврат +1).
- Inline-подсказка под `img-prompt`: «Промпт нарушает правила контента. Переформулируйте.» (`--on-bg-mute`, без морализаторства, без красного алярма).

Модель данных экрана (frontend-state + предполагаемые API-поля).

**Состояние генератора (form):**

**История генераций (`img-grid`) — массив карточек:**
- `image_id: string` (org-scoped);

## 9. Edge cases

- **Очень длинный промпт** (> N символов): поле авто-растёт до 3 строк, далее внутренний скролл; счётчик символов mono (`1240/2000`) у нижнего края поля; в meta/lightbox промпт обрезается «…» с раскрытием. Лимит провайдера на длину — валидация перед submit (toast «Промпт слишком длинный»).
- **0 генераций** — см. §6.1 (empty-state, скрыть grid).
- **Много в истории** (> 8 карточек, десятки/сотни): grid становится скроллируемым; **пагинация/“показать ещё”** или infinite-scroll по 8–16; ПРЕДЛОЖЕНИЕ — группировка по дате («Сегодня / Вчера / Июнь») mono-разделителями; фильтр по style/provider в шапке grid.
- **Медленный провайдер** (> 30 с): карточка остаётся в `loading` с честным ETA/«В очереди…»; через таймаут (напр. 90 с) → auto-error с «Повторить»; пользователь может уйти с экрана — генерация продолжается фоном, результат подтянется (опрос/websocket).
- **Batch**: ×4 списывает −4 лимита сразу; частичный успех (3 из 4) — успешные `done`, упавшие `error` с возвратом лимита поштучно; если остатка лимита < batch_n → batch недоступен (tooltip «Осталось 2 из 4 — уменьшите количество»).
- **Параллельные генерации** из разных вкладок одной орг: лимит сверяется на бэке (server-authoritative), оптимистичный UI откатывается при рассинхроне.
- **Сетевой обрыв во время генерации**: карточка `loading` → при возврате связи ресинк статуса; если результат уже готов на бэке — подтянуть, не тратя повторно.
- **Сброс лимита 1-го числа** в открытой сессии: счётчик и баннер обновляются по `reset_at` без перезагрузки.

## 11. Открытые вопросы / ПРЕДЛОЖЕНИЯ

1. **ПРЕДЛОЖЕНИЕ — Batch ×1 / ×4.** Добавить переключатель количества в `img-controls` (пилюля `▸ Кол-во: 4`). Списывает лимит пачкой. Нужно решение: дефолт 1 или 4; как считать частичный успех (реализовано в Edge cases).
2. **ПРЕДЛОЖЕНИЕ — Reference / эталоны (img2img).** `Paperclip` в `img-prompt` для загрузки картинки-образца + слайдер «сила влияния» (strength). Открытый вопрос: какие провайдеры это поддерживают (gpt-image-2 — да; Pollinations — ограниченно).
3. **ПРЕДЛОЖЕНИЕ — История промптов.** Боковая панель/дропдаун `History` с последними промптами и «закреплёнными» пресетами; повторное применение в один клик. Открытый вопрос: где живёт — drawer справа или поповер.
4. **ПРЕДЛОЖЕНИЕ — Редактирование / inpaint.** В lightbox — режим «дорисовать область» (маска + промпт) и outpaint (расширить холст). Большая фича, отдельный под-экран; помечаю как roadmap.
5. **ПРЕДЛОЖЕНИЕ — Цвет воркспейса в навигации.** Подсветить active-иконку «Картинки» в `--ws-images` (см. §4.1). Решает читаемость воркспейса, но отступает от мокапа (там всё violet). Требует подтверждения владельца.
6. **ПРЕДЛОЖЕНИЕ — Progress-линия лимита** под счётчиком (`--ws-images`, 2px) + цветовые пороги 80%/100%. Требует дизайн-ревью (не перегружает ли тулбар).
7. **Открытый вопрос — Сетка истории при сотнях картинок.** Пагинация vs infinite-scroll vs группировка по датам (§9). Нужен продуктовый выбор.
8. **Открытый вопрос — Хэндофф из Чата.** Как приходит промпт «Запускаю в воркспейсе Картинки»: автозаполнение `img-prompt` + автозапуск? или только заполнение с ожиданием submit? (по умолчанию — заполнить, НЕ запускать без явного действия).
9. **Открытый вопрос — Размеры по провайдерам.** gpt-image-2 и Pollinations поддерживают разные наборы размеров/соотношений → пилюля Size должна динамически фильтроваться под выбранную модель. Нужна карта поддержки.
10. **Открытый вопрос — Где хранятся картинки и retention.** Срок жизни в истории, квота хранилища на тариф, экспорт всей истории (zip). Влияет на §9 «много в истории».
11. **ПРЕДЛОЖЕНИЕ — Сохранить в Документы / Базу знаний.** Действие на карточке «→ В документ» (вставить в A06) / «→ В базу знаний». Сквозная связка воркспейсов.
12. **Открытый вопрос — meta-плашка blur.** Оставить `backdrop-filter: blur(8px)` (как мокап) или заменить на плотный градиент-подложку (строже по «без glass»)? Рекомендация: градиент-подложка в проде.

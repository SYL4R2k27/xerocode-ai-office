# A12 · Empty state · Шаблоны для старта — `/new` (AI-роутер задачи)

> Источник: `XEROCODE_FULL.html` строки 2737–2796. Ключевая идея — **AI-роутер**: пользователь НЕ выбирает воркспейс руками, система определяет воркспейс+модель по тексту задачи. Расширения помечены «ПРЕДЛОЖЕНИЕ».
> Токены — именами CSS-переменных. Иконки — lucide-react. UI — русский. Запрещёнка соблюдается.

## 1. Назначение
«Нулевой экран» продукта: единая точка входа в работу. Юзер описывает задачу естественным языком — AI-роутер подбирает правильный воркспейс (Чат/Текст/Картинки/Код/…) и модель, либо стартует с готового шаблона. Снимает проблему «8 воркспейсов — с чего начать». Используется как `/new` и как **empty-state любого пустого контекста** (в т.ч. пустой чат A03 — переиспользование).

## 2. Доступ
authenticated. Доступен всегда (кнопка «+ Новая задача» в тулбаре/сайдбаре ведёт сюда).

## 4. Регионы и компоненты
- **app-side** — стандартный сайдбар (см. A02), активного воркспейса нет (мы «над» ними).
- **eyebrow** «▸ С чего начать» — mono 11px `--xero` +22% UPPERCASE.
- **H1** «Что сегодня делаем?» — `--display` 500, 44px, `-0.03em`, `--on-bg`, центр.
- **sub** — `--body` 16px `--on-bg-mute`, max-width 460, центр; акцент «AI подберёт правильный воркспейс и модель».
- **task-input** — крупная плашка: `--bg-soft`, border `--line-bg-strong`, radius 14, pad 18×22, flex; placeholder-пример (`--on-bg-dim` 14px); кнопка `→` (pill `--xero`/`--cream`, `ArrowRight`). Слева опц. `+` attach (`Plus`/`Paperclip`) и `🎤` голос (`Mic`) — ПРЕДЛОЖЕНИЕ. Focus → border `--xero`.
- **mono-divider** «▸ ИЛИ ШАБЛОН» — `--on-bg-dim`.
- **template-card** (3 шт): `--bg-soft`, border `--line-bg`, radius 10, pad 16; ico 24px (плашка цвета воркспейса-цели: Email→`--ws-text`, Баннер→`--ws-images`, Отчёт→`--ws-corp`); название (`--display` 500, 14px); тайминг mono (9.5px `--on-bg-dim`, «5 секунд»). Hover → border `--xero`.
- **lucide:** `Sparkles` (eyebrow), `ArrowRight, Plus, Mic`, шаблоны — `Mail, ImagePlus, FileBarChart`.

## 5. AI-роутер (ядро экрана — раскрыто)
- Юзер вводит задачу → отправка → **роутинг**: классификатор определяет воркспейс + модель (+ оценка сложности → одиночный воркспейс vs оркестрация).
- **Показ выбора (ПРЕДЛОЖЕНИЕ):** перед переходом краткий чип-индикатор «→ Воркспейс: Текст · Модель: Claude Opus» с возможностью переопределить (dropdown). Прозрачность > магия.
- **Переход:** плавный (≤300ms) в целевой воркспейс с уже заполненным запросом; исходная формулировка остаётся «инициатором» (как в чат-хэндоффе A03 §7).
- **Многошаговая задача:** если задача комплексная — предложить собрать DAG (мост в A08): «Похоже на повторяемый процесс — автоматизировать?».

## 6. Состояния
- **Чистый старт** (как в мокапе).
- **Ввод** — кнопка `→` активна при непустом тексте.
- **AI-роутинг loading** — без спиннера >200ms: микро-индикатор «Подбираю воркспейс…» (skeleton/точки ≤200ms).
- **Маршрут определён** — чип воркспейс+модель, авто-переход или подтверждение.
- **Неоднозначно** — уточняющий вопрос («Это текст или картинка?») с быстрыми кнопками.
- **Ошибка роутинга** — фолбэк в Чат с введённым текстом.
- **Нет доступа по тарифу** — если AI выбрал Корпоратив/Видео на Free → «доступно на Pro/Enterprise» + апселл, предложить альтернативу.

## 7. Адаптив
≥1280 — центр, шаблоны 3 кол; 768–1279 — шаблоны 3 кол уже/2; <768 — app-side→drawer, task-input на всю ширину, шаблоны 1 кол (стек), H1 меньше.

## 8. Данные
task_text, suggested_templates[], routing_result (workspace+model+confidence), tariff-доступность воркспейсов. Шаблоны — из библиотеки (ПРЕДЛОЖЕНИЕ: персональные/командные/системные).

## 9. Edge cases
пустой ввод (→ disabled); очень общая задача (→ уточнение); задача на недоступный воркспейс (→ апселл); очень длинный ввод (textarea растёт); повтор недавней задачи (ПРЕДЛОЖЕНИЕ: «Недавнее» под инпутом).

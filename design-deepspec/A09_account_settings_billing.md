# A09 · Аккаунт-экосистема — Личный кабинет · Профиль · Настройки · Подписки/Биллинг

> Глубокая дизайн-спецификация. Этой зоны **нет в мокапе** (в `XEROCODE_FULL.html` есть только экран настроек BYOK, строки 2510–2608) — поэтому всё, кроме BYOK-экрана, спроектировано с нуля и согласовано с дизайн-системой v3. Спорные/новые решения помечены «ПРЕДЛОЖЕНИЕ».
> Токены — именами CSS-переменных. Иконки — lucide-react. UI — на русском. Запрещёнка соблюдается (без курсива/serif/больших градиентов/glass/glow/частиц/хайпа/спиннеров >200ms).
> Базовый каркас — `settings-grid` (нав-сайдбар 220px + контент 1fr), всё на Carbon (`--bg`). Источник BYOK-части: реальный мокап A09.

## 1. ЛИЧНЫЙ КАБИНЕТ — `/account`

### 1.1 Назначение
Сводный экран состояния учётной записи: какой тариф, сколько израсходовано (запросы/токены/картинки/DAG-раны), статус BYOK, команда и её seats, ближайший платёж, быстрые действия (апгрейд, пополнить, пригласить). Это «приборная панель аккаунта», в отличие от `/dashboard` (A02) — там приборная панель **работы** (активность по воркспейсам). ПРЕДЛОЖЕНИЕ: вход в кабинет — клик по футер-аватару в любом app-side → выпадает user-menu (§5) → «Личный кабинет».

### 1.2 Доступ
authenticated; токен `ai_office_token`; данные строго по `organization_id` и `user_id`. Видимость блока «Команда» — только владельцу/админу организации (роль). Гость → `/auth`.

### 1.3 Раскладка (широкая дашбордная, НЕ узкая колонка)

              │ └────────────────────────────────────────────────────┘

### 1.4 Регионы и компоненты
- **account-head** — крупная карточка (`--bg-tile`, border `--line-bg`, radius 14): аватар 48px (инициал на `--xero`), имя (`--display` 500, 20px, `--on-bg`), email (`--body` 14px, `--on-bg-mute`), бейдж тарифа (pill, `--xero-mute` фон + `--xero` текст + иконка `Star` 12px для Pro / `Building2` для Enterprise). Справа строка организации + кнопка `btn-mini` «Управлять →» (`Settings` icon). 
- **usage-grid** — 4 `metric`-карточки как в A02, но с **прогресс-баром лимита**: label (mono 10.5px `--on-bg-dim` UPPERCASE), значение (`--display` 500, 28px), полоска (h 4px, фон `--line-bg`, заливка `--xero`; при ≥90% → заливка `--sylar`). Подпись «X% от лимита» или «безлимит» (при BYOK). Иконки: `MessageSquare`/`Coins`/`Image`/`Workflow`.
- **account-row** — 2×2 блоки (`--bg-soft`, border, radius 12, pad 24):
  - **Тариф**: название + цена/мес, дата продления, `btn-mini primary` «Сменить тариф» → `/settings/billing`.
  - **BYOK-ключи**: список 5 провайдеров с dot-статусом (● `--ws-code` активен / ○ `--on-bg-dim` не подключён), счётчик «N из 5», ссылка «Управлять ключами →».
  - **Команда**: «6/10 seats», стопка аватаров (overlap −8px) + «+3», `btn-mini` «Пригласить» (`UserPlus`).
  - **Ближайший платёж**: сумма + дата, маскированная карта (`CreditCard` icon). Если BYOK-only и Free → «Платежей нет».

### 1.5 Поведение и интеракции
- Прогресс-бары: hover → tooltip «4 280 / 6 000 запросов · сброс 1 июля».
- При достижении 100% лимита по любому ресурсу → карточка получает border `--sylar` + кнопка «Снять лимит (BYOK)» / «Повысить тариф».
- «Управлять →» ведёт в соответствующий раздел настроек (роутинг внутри settings-grid).
- Стопка аватаров: клик → `/settings/team`.

### 1.6 Состояния
- **Free без BYOK**: usage-bars близко к лимиту, акцентный апселл-баннер сверху «Подключите свой ключ — безлимит бесплатно» (`--xero-mute` фон, `KeyRound` icon, ссылка).
- **BYOK-only**: значения без баров, подпись «безлимит», бейдж «Свои ключи».
- **Enterprise**: блок «Команда» расширен (роли, домены), бейдж `Building2`.
- **loading**: skeleton карточек (shimmer ≤200ms, без спиннера).
- **Solo (1 seat)**: блок «Команда» сворачивается в «Пригласить команду →».

### 1.7 Адаптив
- ≥1280: usage-grid 4 кол, account-row 2×2.
- 768–1279: usage-grid 2×2, account-row 1 колонка (стек).
- <768: app-side → drawer (гамбургер в топ-баре); всё в один столбец; account-head — аватар над текстом.

### 1.8 Данные
user (имя/email/аватар/язык/роль), organization (название/id/seats used/total), subscription (план/цена/период/дата продления/способ оплаты), usage (по ресурсам: использовано/лимит/сброс), byok-keys (провайдер→статус), team (участники→роль/аватар).

### 1.9 Edge cases / ПРЕДЛОЖЕНИЯ
- Просрочен платёж → красная плашка `--sylar` «Оплата не прошла · обновите карту» (блокировка premium-моделей, не всего).
- Организация на паузе/триал заканчивается через N дней.
- ПРЕДЛОЖЕНИЕ: переключатель организаций, если юзер состоит в нескольких (org-switcher в user-menu).

### 2.3 Регионы и компоненты
- **Фото**: аватар 64px (инициал/изображение), `btn-mini` «Загрузить» (`Upload`), ghost «Удалить». Подпись «PNG/JPG до 2 МБ».

- **Email**: значение + бейдж «✓ подтверждён» (`--ws-code`, `BadgeCheck`) или «не подтверждён» (`--sylar`, кнопка «Отправить письмо»). «Изменить» → инлайн-форма с подтверждением паролём.
- **Язык**: dropdown (Русский/English — ПРЕДЛОЖЕНИЕ, дефолт Русский).
- **Опасная зона**: карточка border `--line-cream`? нет — `border-left: 3px solid --sylar`, заголовок mono `⊘ ОПАСНАЯ ЗОНА` (`--sylar`), кнопка ghost-danger «Удалить аккаунт» → модалка с вводом email для подтверждения.

### 2.4 Состояния / Edge
loading skeleton; сохранение → инлайн «Сохранено ✓» (fade, без тоста-спиннера); ошибка валидации email инлайн под полем; смена email → re-verify flow; удаление → если владелец организации с участниками — блок «сначала передайте владение».

### 2.5 Адаптив
<768: контролы переносятся под label (settings-row → стек), часовой пояс/язык на всю ширину.

## 3. БЕЗОПАСНОСТЬ — `/settings/security`

### 3.1 Состав
- **Пароль**: «Изменить пароль» (текущий + новый + повтор; индикатор силы — полоска `--xero`).
- **2FA / Двухфакторная** (ПРЕДЛОЖЕНИЕ): тумблер; методы — TOTP-приложение (QR), Telegram-код. Бейдж «Включена» (`ShieldCheck` `--ws-code`).
- **Активные сессии**: список устройств (браузер/ОС/город/последняя активность/IP-маска), текущая помечена «● сейчас», кнопка «Завершить» на остальных, «Завершить все».
- **Журнал входов** (ПРЕДЛОЖЕНИЕ): таблица последних входов (время/устройство/результат).
- **OAuth-привязки**: Google / Telegram — «Привязано»/«Привязать» (из A01 OAuth).

### 3.2 Компоненты/токены
session-row: иконка устройства (`Monitor`/`Smartphone`), bg `--bg-soft`, текущая — border-left `--xero`. Кнопки завершения — ghost-danger. Тумблер 2FA — `--xero` в active. Иконки: `Lock`, `ShieldCheck`, `KeyRound`, `LogOut`.

### 3.3 Состояния/Edge
нет 2FA → акцентная подсказка включить; одна сессия → «Завершить все» скрыто; смена пароля → разлогинить прочие сессии (чекбокс).

**Компоненты:**

### 4.2 Модели по умолчанию — `/settings/models` (ПРЕДЛОЖЕНИЕ)
Назначение каждому воркспейсу дефолт-модели и фолбэк-цепочки. Таблица: воркспейс → основная модель (dropdown) → фолбэк. Тумблер «разрешить платный фолбэк» (`allow_paid_fallback` из бэка). Тумблер «использовать BYOK в первую очередь».

### 4.3 Лимиты — `/settings/limits`
Просмотр/настройка месячных лимитов по ресурсам (запросы/токены/картинки/DAG). Для Enterprise — лимиты на участника команды. Бары как в кабинете. ПРЕДЛОЖЕНИЕ: алерты при 80%/100% (email/Telegram).

### 4.4 История запросов — `/settings/history`
Таблица: время / воркспейс / модель / токены / стоимость / статус. Фильтры (воркспейс, модель, период). Экспорт CSV. Связь с биллингом (расход).

**Гейтинг (`plans.py`):** воркспейс `corp` и CORP-модули (`crm/channels/team_tasks/hr/edo/integrations_*/rbac_roles`) — только Enterprise (RBAC 5 ролей) и Enterprise+ (RBAC ∞ + ЭДО + WhatsApp/VK). Годовая оплата −15…25%. Legacy-имена планов (`corporate→enterprise`, `pro_plus→prime`) маппятся в `get_plan()`.

### 6.4 Компоненты/токены
- **current-plan-banner**: фон `--xero-mute`, border `--xero`, radius 12; план + цена + seats + кнопки `btn-mini` «Управлять»/ghost-danger «Отменить».
- **plan-card**: `--bg-soft`, border `--line-bg`; **текущий** — border `--xero` + бейдж «Ваш план»; **рекомендуемый** — лента mono «▸ ПОПУЛЯРНЫЙ». Заголовок (`--display` 500, 18px), цена (`--display` 500, 28px + «/мес» mute), список фич (✓ `--ws-code` / — `--on-bg-dim`), CTA pill (primary для апгрейда, ghost для текущего/контакта). Иконки: `Star` (Pro), `Building2` (Enterprise), `Check`, `Minus`.
- **byok-callout**: акцентная полоса с `KeyRound`, ссылка в `/settings/api-keys`.
- **payment-method**: карта (`CreditCard`), методы РФ — **Карта МИР/Visa/MC, СБП**, для юрлиц «Выставить счёт» (`FileText`). ФЗ-54: после оплаты — чек на email.

### 6.5 Поведение / флоу
- **Апгрейд** Free→Pro: модалка → выбор периода (мес/год −20%) → способ оплаты → подтверждение → success (мгновенный анлок воркспейсов). 
- **Даунгрейд**: предупреждение что отключится (воркспейсы/seats сверх лимита) → действует с конца оплаченного периода.
- **Отмена**: «останется活ным до 28.06, потом Free». Причина (опрос — ПРЕДЛОЖЕНИЕ).
- **Enterprise**: не самообслуживание — «Связаться» → форма/Telegram, договор, безнал.
- **Seats**: добавление seat в Pro/Enterprise — пропорциональная доплата (proration).

### 6.6 Состояния / Edge
триал-баннер (осталось N дней); платёж не прошёл → `--sylar` баннер + retry; годовая скидка; промокод (поле «Применить промокод» — ПРЕДЛОЖЕНИЕ); валютный/НДС-нюанс для юрлиц РФ; апгрейд в момент исчерпания лимита (контекстный CTA из кабинета/воркспейса).

## 10. Главные открытые вопросы (на обсуждение с Claude design)
1. **Цены/состав тарифов** — закрепить числа (1 490 ₽ Pro? порог Enterprise?), что именно гейтится за Pro vs Enterprise (Корпоратив — точно Enterprise по мокапу).
2. **`/account` vs `/dashboard`** — нужны ли оба, или кабинет = вкладка дашборда? (ПРЕДЛОЖЕНИЕ: разделить — работа vs учётка).
3. **BYOK-провайдеры** — показывать ли все 10 из бэка или только 5 из мокапа + «Ещё».
4. **Роли команды** — сколько ролей, права матрицей.
5. **Биллинг РФ** — самообслуживание для физлиц (карта/СБП) + счета/акты/УПД для юрлиц; ФЗ-54 чеки; НДС.
6. **2FA/SSO** — объём для SMB vs Enterprise.
7. **Тема dark/light** — где переключатель, и нужен ли light вообще (сейчас Cream только в A06).

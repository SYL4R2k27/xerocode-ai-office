# A11 · Onboarding — `/onboarding` (first-time user)

> Источник: `XEROCODE_FULL.html` строки 2656–2735 (точно показан только ШАГ 2). Шаги 1/3/4 спроектированы из системы — помечены «ПРЕДЛОЖЕНИЕ».
> Токены — именами CSS-переменных. Иконки — lucide-react. UI — русский. Запрещёнка соблюдается.

## 1. Назначение
Первичная настройка за ~2 минуты сразу после регистрации: персонализировать интерфейс под workflow (какие воркспейсы показывать), подключить ключи/модели, запустить первую задачу. Цель — быстрый «aha»: пользователь выходит из онбординга в готовый к работе кабинет.

## 2. Доступ
authenticated, first-run flag. Можно «Пропустить» (ПРЕДЛОЖЕНИЕ) → дефолтный набор воркспейсов. Повторно — из `/settings`.

## 4. Регионы и компоненты
- **progress-bar:** 4 сегмента `flex:1, h:3px, radius:2px`; пройдено/текущий — `--xero`, остальное — `--line-bg-strong`. Анимация заливки при переходе шага (`--ease`, ≤300ms).
- **eyebrow:** mono 11px, `--xero`, +22% tracking, UPPERCASE, маркер «▸», + длительность шага.
- **H1:** `--display` 500, `clamp(36px,4.5vw,64px)`, line-height 1, `-0.03em`, `--on-bg`.
- **sub:** `--body` 16.5px, `--on-bg-mute`, max-width 540.
- **ws-card** (8 шт, выбираемая): pad 22×18, radius 12.
  - невыбранная: фон `--bg-soft`, border `--line-bg`; ico 32px (плашка цвета воркспейса `--ws-*`, radius 8); название (`--display` 500, 17px); краткое описание (`--body` 12px `--on-bg-mute`).
  - выбранная: фон `--xero-mute`, border `--xero`; вместо описания — mono «▸ ВЫБРАНО» (9.5px, `--xero`, +18% tracking). Иконка-галка `Check` (ПРЕДЛОЖЕНИЕ) в углу.
- **footer:** `← Назад` (ghost pill, border `--line-bg-strong`), счётчик mono «ВЫБРАНО N ИЗ 8» (`--on-bg-dim`), `Дальше →` (primary pill `--xero`/`--cream`).
- **lucide:** воркспейсы — `MessageSquare, FileText, Image, Code2, Video, AudioLines, Building2, Workflow`; навигация — `ArrowLeft, ArrowRight, Check`.

## 5. Полный флоу 4 шага (1/3/4 — ПРЕДЛОЖЕНИЕ)
1. **Знакомство** — имя/роль/размер команды (solo vs команда), чтобы подобрать дефолты и тариф-подсказку. Поля: «Как вас зовут», «Чем занимаетесь» (чипы: разработка/маркетинг/продажи/контент/менеджмент), «Работаете один или с командой».
2. **Воркспейсы** (есть в мокапе) — выбор 2–3 главных; влияет на состав app-side.
3. **Ключи / модели** (ПРЕДЛОЖЕНИЕ) — «Подключить свой ключ (BYOK = ∞)» с быстрыми кнопками OpenAI/Anthropic/Yandex/GigaChat, либо «Пока на бесплатных моделях». Объясняет ценность BYOK на входе.
4. **Первая задача** (ПРЕДЛОЖЕНИЕ) — мост в A12: «Опишите первую задачу» или выбор шаблона → сразу в нужный воркспейс. Финальная кнопка «Перейти в XeroCode →».

## 6. Поведение и интеракции
- Тап карточки — toggle выбора (scale-pulse ≤180ms). Мин 1, рекоменд. 2–3, макс 8.
- «Дальше» disabled, пока не выбран минимум (на шаге 2 — ≥1). Disabled-кнопка: `--on-bg-dim` текст, `--bg-soft` фон, cursor not-allowed.
- «Назад» сохраняет состояние шага. Прогресс кликабелен только по пройденным шагам.
- «Пропустить» (ПРЕДЛОЖЕНИЕ, мелкая ссылка вверху-справа) → дефолт (Чат+Текст+Картинки+Код) и сразу в `/dashboard`.

## 7. Состояния
- 0 выбрано → «Дальше» disabled + подсказка «Выберите хотя бы один».
- все 8 → ок (но подсказка «можно меньше — остальные всегда доступны»).
- переход шага → progress анимируется, контент cross-fade.
- обновление страницы → прогресс сохранён (draft на бэке/локально).

## 8. Адаптив
≥1280 grid 4×2; 768–1279 grid 2×4; <768 grid 1 кол (список карточек), footer липкий снизу (`position: sticky/fixed`, фон `--bg` + hairline сверху), H1 уменьшается clamp.

## 9. Данные
profile (имя/роль/тип), selected_workspaces[], byok_choice, first_task, onboarding_step/completed. Влияет на app-side состав, дефолт-модели, апселл-логику.

## 10. Edge cases / Маппинг
- Уже онбордился → редирект в `/dashboard`.
- Выбрал Корпоратив на Free → пометка «доступно на Enterprise», не блокируем выбор (покажем апселл позже).
- **Компоненты:** `OnboardingShell`, `ProgressBar`, `SelectableCard`, `StepRouter`. CSS-переменные, lucide.

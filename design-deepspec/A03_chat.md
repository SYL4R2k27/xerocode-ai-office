# A03 · Чат с AI — `/chat`

> Глубокая дизайн-спецификация приоритетного экрана XeroCode v3.
> Источники правды: `XEROCODE_REDESIGN_BRIEF.md` §6/§3, `XEROCODE_FULL.html` строки 1987–2110 (точный мокап), `xerocode-v3.css` строки 875–1044 (точные стили), V2-компоненты `chat/*`, `layout/SidebarV2`.
> Все цвета — **именами CSS-переменных**, все иконки — **lucide-react**, весь UI — **на русском**.
> Запрещёнка (соблюдается во всём документе): без курсива/serif/editorial-хрома/градиентных заливок на больших площадях/glassmorphism/glow/neon/3D-blobs/particles/спиннеров дольше 200ms.

---

## 1. Назначение

Воркспейс **01 · Чат** (`--ws-chat #7C5CFF`) — главная точка работы с языковыми моделями. Единый чат-интерфейс поверх **14 моделей** (Premium / Standard / РФ / Free), с историей диалогов, ветвлением ответов (branching), цитированием источников (citations), переносом контекста между воркспейсами и хэндоффом артефактов в другие воркспейсы (Documents, Images, Code…).

Ядро ценностного предложения продукта *«один человек делает работу пятерых»* реализуется именно здесь: пользователь не выбирает провайдера руками на уровне API — он формулирует задачу, выбирает модель в правой панели (или доверяет дефолту), а система роутит запрос через BYOK-ключи (`use_proxy = bool(proxy)`).

**Чего экран НЕ делает:** не редактирует документы (это A06), не генерирует картинки в гриде (A04), не строит DAG (A08). Эти задачи он **передаёт** соответствующим воркспейсам через хэндофф (§7.7) — но переписка-инициатор остаётся здесь как «журнал решения».

## 2. Доступ

| Параметр | Значение |
|---|---|
| URL | `/chat` (в мокапе url-бар: `app.xerocode.ru / chat`) |
| Аутентификация | **Только authenticated.** Гость → редирект на `/auth` (A01) |
| Токен | `ai_office_token` в localStorage (единый ключ проекта) |
| Изоляция | Все диалоги/сообщения фильтруются по `organization_id` текущего юзера (правило проекта №7) |

| Компоненты | V2: `SidebarV2`, `ChatAreaV2`, `ChatInputV2`, `ChatMessageV2` + **новый** `ChatSidePanelV2` (right-rail) |

## 4. Регионы и компоненты

### 4.1 Колонка 1 — `chat-list` (история диалогов) · 280px · `--bg-soft`

**Контейнер:** `padding: 20px 14px`, `border-right: 1px solid var(--line-bg)`. В мокапе `overflow-y: hidden` — на проде заменить на `overflow-y: auto` со скрытым скроллбаром (`scrollbar-width: none`, hairline-индикатор справа на hover).

**4.1.1 Поиск (`.search`)**
- Псевдо-инпут: `--bg`, `border 1px var(--line-bg)`, `radius 8px`, `padding 10px 14px`.
- Текст: `--mono`, `11.5px`, цвет `--on-bg-dim`. Иконка `Search` (lucide, 13px, `--on-bg-dim`) слева вместо emoji 🔍 из мокапа (**правило: иконки только lucide**). Плейсхолдер: «Поиск по диалогам…».
- `margin-bottom: 14px`.
- Состояние focus: `border-color: var(--line-bg-strong)`; при вводе — реальный `<input>` с тем же стилем, текст становится `--on-bg`.

**4.1.2 Элемент диалога (`li`)**
- `padding 12px 14px`, `radius 8px`, `flex-column gap 4px`, `margin-bottom 2px`, `transition all 180ms var(--ease)`.
- **`.nm`** (название): `--display` нет — в CSS это `13.5px / weight 500 / --on-bg`, одна строка с `ellipsis`. Пример: «Презентация инвесторам».
- **`.pre`** (превью первой строки): `12px / --on-bg-mute`, одна строка `ellipsis`. Пример: «Сегодня тебе нужно подготовить…».
- **`.ts`** (время): `--mono / 9.5px / --on-bg-dim / letter-spacing .05em`. Форматы: `14:32` (сегодня), `Вчера`, `2 дня`, `12 мая` (>7 дней). Позиция — в потоке под `.pre` (в мокапе так), не absolute.
- **hover:** `background: rgba(244,239,227,0.04)`.
- **active:** `background: var(--xero-mute)` (без бордера — в мокапе только фон). ПРЕДЛОЖЕНИЕ ниже — добавить левый violet-маркер 2px для усиления.

> **ПРЕДЛОЖЕНИЕ (active-маркер):** активный диалог — добавить `box-shadow: inset 2px 0 0 var(--xero)` (левая полоса 2px). Не нарушает «минимум теней» (это border-имитация, не drop-shadow), резко улучшает сканируемость списка из 50+ элементов.

**4.1.3 Кнопка «+ Новый чат»**
В мокапе отсутствует. Необходима.
- **ПРЕДЛОЖЕНИЕ A (рекомендую):** sticky-кнопка вверху, **над** поиском: pill-ghost на всю ширину, `Plus` 14px + «Новый чат», `--mono uppercase`, hairline-бордер. Всегда видна.
- **ПРЕДЛОЖЕНИЕ B:** floating-кнопка внизу колонки (`Plus`, круглая 40px, `--xero` заливка). Конфликтует с длинным списком.
- **ПРЕДЛОЖЕНИЕ C:** только хоткей `⌘N` + пункт в шапке app-toolbar.
→ Выбор A: предсказуемое место, не перекрывает контент, соответствует «без floating-хайпа».

**4.1.4 Группировка списка (при 50+ диалогах)**
Mono-разделители (`--mono 10px uppercase --on-bg-dim`, +22% tracking, маркер `▸`): `▸ СЕГОДНЯ`, `▸ ВЧЕРА`, `▸ 7 ДНЕЙ`, `▸ РАНЬШЕ`. Sticky на скролле. См. §9 (edge cases).

### 4.2 Колонка 2 — `chat-area` (переписка) · 1fr · `--bg`

`display: flex; flex-direction: column`. Три горизонтальных региона: head / messages / input.

**4.2.1 `chat-head` (шапка диалога)**
- `padding 18px 28px`, `border-bottom 1px var(--line-bg)`, `flex space-between center`.
- **`.title`**: `--display / weight 500 / 17px / letter-spacing -0.015em / --on-bg`. Пример: «Презентация инвесторам». Inline-edit по двойному клику (ПРЕДЛОЖЕНИЕ).
- **`.meta`** (справа): `--mono / 10.5px / +0.18em tracking / UPPERCASE / --on-bg-dim`, `flex gap 14px`. Структура: `[Имя модели] · [N токенов]`. Пример: `CLAUDE OPUS 4.7 · 4 580 ТОКЕНОВ`.
- **ПРЕДЛОЖЕНИЕ (действия head):** справа от меты — два icon-button 28px (`--on-bg-dim`, hover `--on-bg`): `MoreHorizontal` (меню: Переименовать / Дублировать / Экспорт MD / Архивировать / Удалить) и опционально `Share2`. Иконки появляются на hover всей шапки, чтобы не шуметь.

**4.2.2 `chat-messages` (лента)**
- `flex 1`, `padding 24px 28px`, `gap 18px`, `flex-column`. На проде `overflow-y: auto` (в мокапе hidden), auto-scroll к низу при новом сообщении (уже есть в `ChatAreaV2`: `scrollRef.scrollTop = scrollHeight`).
- **`.msg`** — общий: `flex gap 12px`, `max-width 80%`. Аватар + тело.
- **Аватар `.av`**: круг 28px, `--display / weight 600 / 11px / цвет --cream`.
  - `.av.ai` → фон `--xero`, текст «AI».
  - `.av.me` → в мокапе `linear-gradient(135deg, var(--ws-text), var(--ws-images))`. **Внимание: это градиент.** Допустимо как 28px-аватар (мелочь, не «большая площадь»), но если строго трактовать запрет — **ПРЕДЛОЖЕНИЕ:** заменить на плоский `--steel` фон с инициалом «V». Решение за владельцем (см. §11 откр. вопрос).
- **`.msg.me` (юзер):** `align-self: flex-end`, `flex-direction: row-reverse` (аватар справа). Тело: фон `--xero`, текст `--cream`, бордер `--xero`. Пример: «Сделай 10-страничную презентацию…».
- **`.msg` (ai):** аватар слева. Тело: фон `--bg-soft`, бордер `1px --line-bg`, `radius 12px`, `padding 12px 16px`, `14px / line-height 1.55 / --on-bg`.
- **Inline `<code>`** внутри `.body`: `--mono / 12.5px`, фон `--bg`, `padding 2px 6px`, `radius 4px`, бордер `1px --line-bg`. Пример: `Documents`. (Это inline-код; блочный код — §6.7.)
- **`.src` (citation):** внутри `.body` снизу. `--mono / 10px / +0.18em / UPPERCASE / цвет --xero`, `margin-top 8px`, `padding-top 8px`, `border-top 1px --line-bg`. Маркер `▸`. Пример: `▸ SOURCE · ШАБЛОН Y-COMBINATOR DECK · Q1 2026 SYLAR DATA`. Поведение — §6.4.
- **`<br>`-структура в мокапе:** AI перечисляет «1. Cover … 10. Ask» через `<br>`. На проде это рендерится из Markdown (`ReactMarkdown` + `remarkGfm`, уже в `ChatMessageV2`) как `<ol>` со своим ритмом — не хардкодить `<br>`.

**4.2.3 `chat-input`**
- `padding 16px 28px 20px`, `border-top 1px --line-bg`.
- **`.chat-input-box`**: `--bg-soft`, бордер `1px --line-bg-strong` (сильнее, чем у сообщений — поле ввода важнее), `radius 12px`, `padding 14px 16px`, `flex center gap 12px`, `--body / 13.5px / --on-bg-dim` (плейсхолдер).
- Плейсхолдер: «Опишите задачу или прикрепите файл…».
- **`.send`**: квадрат 32px, `radius 8px`, фон `--xero`, текст `--cream`, `margin-left auto`. В мокапе символ `→`; на проде — `Send` (lucide, 16px). Во время стрима → `Square` (стоп). Disabled (пусто) → `--steel` фон, не кликабельна.
- **Слева:** кнопка attach `Plus`/`Paperclip` (lucide 18px, `--on-bg-dim`) — открывает меню вложений (§6.3). Опц. `Mic` для голосового ввода (есть в `ChatInputV2`).
- **Под полем (ПРЕДЛОЖЕНИЕ):** строка-«футер» инпута, `--mono 9.5px --on-bg-dim`: слева token-индикатор `⌥ 1 240 / 200 000 ток` (§6.5), справа подсказка `⌘↵ — отправить`. Высота 16px, не съедает воздух.

## 6. Поведение и интеракции

### 6.0 Базовые интеракции по элементам

| Элемент | Действие | Реакция |
|---|---|---|
| `li` диалога | click | загрузить диалог в `chat-area`, подсветить active, сбросить scroll наверх (есть в `ChatAreaV2`) |
| `li` диалога | hover | фон `rgba(cream,.04)`, проявить icon-меню (ПРЕДЛ.) |
| `.search` | type | live-фильтр списка (debounce 150ms), подсветка совпадений |
| `.model-pick` | click | сменить модель **для текущего/следующего сообщения**, обновить `.meta` в `chat-head` |
| `.chat-input-box` | focus | бордер усиливается до `--line-bg-strong`→`--xero` (200ms) |
| `.send` | click / `⌘↵` | отправить, очистить поле, добавить `.msg.me`, начать стрим AI |
| `.src` | click | раскрыть/перейти к источнику (§6.4) |
| `+ Документ из Knowledge` | click | открыть attach-picker (§6.3) |

Все переходы — `var(--ease) cubic-bezier(0.22,1,0.36,1)`, длительность 180–300ms. Spinner'ы запрещены дольше 200ms — для ожиданий >200ms использовать **точечный пульс** (3 точки `--on-bg-dim`, opacity-анимация) либо typing-каретку (§6.6).

### 6.1 BRANCHING / Ветки ответов  ⭐ (нет в мокапе → проектируется)

**Модель данных.** У каждого AI-сообщения есть `parent_message_id`. Регенерация или ручное «ответить иначе» создаёт **сиблинг** с тем же `parent_message_id`. Набор сиблингов = ветки. Активная ветка хранится в `thread.active_path` (массив id выбранных сообщений). Дочерние сообщения наследуют ветку родителя — переключение ветки выше по треду **переписывает хвост** ленты ниже.

**Где живёт UI переключателя — варианты:**

- **A. Inline-степпер под сообщением (рекомендую).** Под телом AI-сообщения, рядом с действиями: `ChevronLeft  ‹ 2 / 3 ›  ChevronRight` (`--mono 10px --on-bg-mute`). Клик по стрелкам переключает сиблинг → лента ниже перерисовывается под выбранную ветку (fade 150ms). Компактно, стандартно (как ChatGPT/Claude), не требует отдельной зоны. Появляется только если сиблингов >1.
  ```
  [AI] ┌─────────────────────────┐
       │ …ответ вариант 2…       │
       │ ▸ SOURCE · …            │
       └─────────────────────────┘
       ‹ 2 / 3 ›   [↻ Перегенерировать]   [⧉ Копир.]   [👍][👎]
  ```
- **B. Мини-карта диалога (tree).** В `chat-side` под «Контекст» — вертикальное дерево узлов с развилками (точки + violet-edges). Клик по узлу → прыжок. Мощно для длинных ветвлений, но сложно и шумно для 90% случаев. ПРЕДЛОЖЕНИЕ: дать как **опциональный** оверлей по кнопке `GitBranch` в head, не постоянный.
- **C. Горизонтальные табы вариантов** над телом сообщения (`Вариант 1 | 2 | 3`). Занимает вертикаль, плохо при 4+ ветках.

→ **Выбор: A как основной + B как опциональная карта** (кнопка `GitBranch` 14px в `chat-head`, открывает right-rail-оверлей с деревом; на мобиле — bottom-sheet).

**Как ветвить от regenerate.**
1. Под AI-сообщением — `[↻ Перегенерировать]` (`RotateCcw` 14px + текст, `--on-bg-mute`, hover `--on-bg`).
2. Клик → дропдаун: «Той же моделью» / «Другой моделью ▸» (подменю = текущий model-picker) / «Изменить тон ▸» (Короче/Подробнее/Формальнее/Проще).
3. Новый ответ создаётся как **новая ветка** (сиблинг), степпер сразу показывает `‹ N / N ›` (последняя). Старый вариант не теряется.
4. Если у юзера было **отредактировано своё сообщение** (edit `.msg.me`) — это тоже ветвление: правка создаёт новую пользовательскую ветку, AI отвечает в ней. Под `.msg.me` тот же степпер `‹ 1 / 2 ›`.

**Состояния степпера:** `disabled`-стрелка на краях (opacity .35). При генерации новой ветки — счётчик показывает «генерация» (typing-точки вместо номера до завершения).

### 6.2 MODEL-PICKER на 14 моделей  ⭐

В правую колонку 280px не влезут 14 карточек по 56px (это >780px). Нужна свёртка.

**Полный пул 14 (пример, синхронизировать с `providers_status.md`):**

| Группа | Модели | tier-метка |
|---|---|---|
| **Premium** | Claude Opus 4.7 (200k) · GPT-5.4 (1M) · Gemini 2.5 Pro (1M) · Grok-4 (256k) | `PREMIUM` (violet) |
| **Standard** | Claude Sonnet 4.6 (200k) · GPT-5.4 mini · DeepSeek V3 (128k) · Llama 4 Maverick | `STANDARD` |
| **РФ** | Yandex GPT (32k) · GigaChat Pro (32k) · GigaChat Lite (Free) · T-bank (T-pro) | `РФ` |
| **Free** | Pollinations-text · OpenRouter-free-pool | `FREE` |

**Решение раскладки right-rail (рекомендую — комбинированное):**

- В `chat-side` показывать **3–4 «закреплённых/недавних»** карточки `.model-pick` (как в мокапе: активная + частые). Над ними — `h4 ▸ МОДЕЛЬ`.
- **Кнопка «Все модели · 14»** (pill-ghost, `--mono uppercase`, иконка `LayoutGrid`/`ChevronDown`) → открывает **командную палитру** (см. ниже).
- Хоткей **`⌘K`** открывает ту же палитру из любого места.

**Командная палитра моделей (⌘K) — A/B/C:**

- **A. Центральный модал-командер (рекомендую).** Overlay 520px по центру, `--bg-soft`, бордер `--line-bg-strong`, radius 14px. Сверху — поле поиска (`Search`, `--mono`, плейсхолдер «Поиск модели… (название, провайдер, окно)»). Ниже — список, сгруппированный аккордеонами `▸ PREMIUM / ▸ STANDARD / ▸ РФ / ▸ FREE`. Каждая строка: `.nm` + tier + окно + цена-индикатор + иконка-«закрепить» (`Pin`). Стрелки ↑↓ навигация, `↵` выбор, `Esc` закрыть. Закреплённые всплывают наверх отдельной секцией `▸ ЗАКРЕПЛЁННЫЕ`. **Фильтр-пиллы** под поиском: `Все · Premium · Standard · РФ · Free · Доступны мне` (последний — только те, на что есть BYOK-ключ/лимит).
- **B. Раскрывающийся inline-аккордеон** прямо в right-rail (группы сворачиваются/разворачиваются). Проще, но при 14 моделях колонка становится очень длинной → скролл конфликтует с блоком «Контекст».
- **C. Dropdown из `chat-head`** (имя модели становится кликабельным селектом). Компактно, но прячет богатую мету (окно/цена/доступность).

→ **Выбор: A (⌘K-командер) + 3 закреплённых в rail + кнопка «Все 14».** Покрывает поиск, группы, закреплённые, доступность. Минимум визуального шума в дефолте.

**Метки доступности:** модель без ключа/лимита — `.nm` приглушён (`--on-bg-dim`), бейдж `Нужен ключ` (`--sylar`), клик → ведёт в `/settings/api-keys`. Модель за тарифом (Premium на Free-плане) — бейдж `Pro` (`--xero-mute`).

**Цена-индикатор (ПРЕДЛОЖЕНИЕ):** в строке палитры — точки стоимости `··· ` (1–3 точки `--steel`, как «$/$$/$$$»), чтобы быстро отличать дорогие Premium от дешёвых. Без чисел в rail; числа — в тултипе.

### 6.3 ATTACH / Контекст  ⭐

**Триггеры:** (1) кнопка `Plus`/`Paperclip` в `chat-input` слева; (2) карточка «+ Документ из Knowledge» в `chat-side`; (3) drag&drop файла на `chat-area` (overlay уже реализован в `ChatAreaV2`: «Перетащи файлы сюда»).

**6.3.1 Меню вложений** (поповер над кнопкой attach, `--bg-soft`, hairline, radius 10px):
| Пункт | Иконка | Что цепляет |
|---|---|---|
| Файл с устройства | `Paperclip` | png/jpg/pdf/docx/txt/md (есть `getFileIcon` в `ChatInputV2`) |
| Документ из Knowledge | `BookOpen` | RAG-источник из A07 (Confluence/1С/Notion/SharePoint/Диадок…) |
| Ссылка / URL | `Link` | URL → бэкенд фетчит и кладёт в контекст |
| Прошлый диалог | `MessageSquare` | другой thread как контекст («продолжи мысль из …») |

**6.3.2 Picker «Документ из Knowledge»:** модал-список источников KB (как `kb-results` в A07): src-бейдж коннектора + h4 + meta. Поиск сверху. Множественный выбор (чекбоксы). «Прикрепить (N)».

**6.3.3 Picker «Прошлый диалог»:** переиспользует `chat-list` UI в модале (поиск + список nm/pre/ts), single/multi-select.

**6.3.4 Отображение прикреплённого (чипы):**
- В `chat-input` над полем — ряд чипов: иконка типа + имя (ellipsis) + размер + `X` (удалить). Стиль чипа: `--bg`, hairline, radius 8px, `--mono 10px`. (Паттерн уже есть в `ChatInputV2` для файлов.)
- Дублируются в `chat-side` → блок «Контекст»: каждый attach как `.model-pick`-карточка с `.nm` = имя, `.meta` = тип/размер/источник + крестик удаления. Так пользователь видит «что сейчас в контексте диалога» постоянно, даже после отправки.
- **ПРЕДЛОЖЕНИЕ (единый «+ Контекст»):** объединить все 4 типа под одной кнопкой `+ Контекст` в rail (вместо только «Документ из Knowledge»). Внутри — табы Файл/Knowledge/URL/Диалог. Чище, чем 4 отдельные точки входа.

**6.3.5 Связь attach ↔ сообщение:** прикреплённый контекст применяется к **следующему** отправленному сообщению и остаётся «липким» для треда (можно открепить). В пузыре `.msg.me` под текстом — мелкая строка `▸ 2 вложения` (mono), раскрывается в список.

### 6.4 CITATIONS «▸ Source»  ⭐

**Формат (из мокапа):** `▸ SOURCE · ШАБЛОН Y-COMBINATOR DECK · Q1 2026 SYLAR DATA` — `--mono 10px --xero UPPERCASE`, отделён `border-top 1px --line-bg`, `padding-top 8px`.

**Кликабельность — A/B/C:**
- **A. Раскрывающаяся панель (рекомендую).** Клик по `.src` → разворачивается под сообщением блок-аккордеон со списком источников: каждый = иконка типа (`FileText`/`BookOpen`/`Link`/`Globe`) + заголовок + сниппет-цитата (2 строки, `--on-bg-mute`) + `ExternalLink` (открыть оригинал в новой вкладке/в KB). Анимация высоты 200ms.
- **B. Поповер по hover** — быстрый превью сниппета, но плохо для тача/мобайла.
- **C. Сразу ссылка** (без раскрытия) — теряем сниппет и мульти-источник.

→ **Выбор A.** Поддержка **нескольких** источников: `▸ SOURCE · 3` → раскрывает три. Нумерованные инлайн-маркеры `[1]`,`[2]` в теле ответа (ПРЕДЛОЖЕНИЕ) — кликабельны, скроллят к соответствующему источнику в раскрытом блоке (паттерн Perplexity, но в монохроме бренда: маркер `--xero`, `--mono 10px`, верхний индекс).

**Типы источников:** KB-документ (→ открыть в A07), загруженный файл (→ просмотр), URL (→ внешняя вкладка с confirm для незнакомых доменов), шаблон/системный (→ только текст, без ссылки).

### 6.5 TOKEN-СЧЁТЧИК  ⭐

Три уровня отображения:

1. **Per-thread (в `chat-head .meta`)** — `4 580 ТОКЕНОВ` (из мокапа). Это суммарный контекст текущего диалога. Растёт по мере переписки.
2. **Per-input (под `chat-input`, ПРЕДЛОЖЕНИЕ)** — `⌥ 1 240 / 200 000 ток` — оценка токенов текущего черновика + вложений против **окна контекста выбранной модели**. Обновляется при вводе (debounce). Цвета порога:
   - ≤75% окна — `--on-bg-dim`.
   - 75–95% — `--sylar` (предупреждение) + иконка `AlertTriangle` 12px + тултип «Близко к лимиту окна Opus 4.7 (200k). Старые сообщения могут быть усечены».
   - >100% — блок отправки невозможен «как есть» → предложение «Сжать контекст» / «Сменить на модель с большим окном (GPT-5.4 · 1M)».
3. **Бюджет/лимит тарифа (ПРЕДЛОЖЕНИЕ, в `chat-side` низ или app-side)** — для не-BYOK: `Токены сегодня · 38K / 100K` с тонкой hairline-полосой прогресса (высота 2px, `--xero` заливка по проценту, фон `--line-bg`). При BYOK — вместо полосы бейдж `BYOK · ∞` (`--xero-mute`). При исчерпании лимита — баннер (§8.5).

**Per-message стоимость:** `ChatMessageV2` уже имеет `tokens_used` и `cost_usd`. Показывать опционально в hover-футере сообщения как `mono 9.5px`: `420 ток · $0.012` (для admin/прозрачности; рядовому юзеру — скрыто за тогглом «показывать стоимость» в настройках).

### 6.6 СТРИМ-ГЕНЕРАЦИЯ  ⭐

**Старт:** после отправки — мгновенно (≤200ms) появляется AI-пузырь с **typing-индикатором**: 3 точки `--on-bg-dim`, плавный пульс opacity (`.4→1`, stagger 120ms). Никаких крутящихся спиннеров.

**Стрим текста:** токены добавляются в `.body` инкрементально (markdown рендерится прогрессивно через `ReactMarkdown`; незакрытые код-блоки рендерятся как plain до закрытия ``` ). Каретка `▍` (`--xero`, мигание 1s) в конце текущего текста. Auto-scroll держит низ в зоне видимости (но не дёргает, если юзер проскроллил вверх — тогда показать pill «↓ Новое» внизу).

**Стоп:** кнопка `.send` → `Square` (стоп). Клик или `Esc` (глобальный хоткей уже в `ChatInputV2` и `ChatAreaV2` через `onStop`) — обрывает поток. Частично сгенерированный текст **сохраняется** с пометкой `▸ ОСТАНОВЛЕНО` (mono `--on-bg-dim`) и кнопкой «Продолжить» (досгенерировать) рядом с «Перегенерировать».

**Во время стрима:** инпут активен (можно печатать следующее), но отправка заблокирована до завершения/стопа (или ставит в очередь — ПРЕДЛОЖЕНИЕ). Model-picker во время стрима — disabled (смена модели применится к следующему сообщению).

**Частичный рендер блоков:** таблицы/списки/код появляются по мере парсинга; высота пузыря растёт плавно (без layout-jump — резервировать `min-height`).

### 6.7 КОД-БЛОК (длинный код)  ⭐

`ChatMessageV2` уже использует `react-syntax-highlighter` + `oneDark`. Спецификация обёртки:
- **Шапка код-блока:** `--bg`, hairline-низ, `--mono 10px --on-bg-dim`: слева — язык (`PYTHON`), справа — `[⧉ Копировать]` (`Copy`→`Check` 14px) + `[⌄ Свернуть]` (`ChevronDown`/`ChevronRight`).
- **Тело:** `--mono 12.5px`, нумерация строк (`--on-bg-dim`), горизонтальный скролл при длинных строках (не перенос). Тема oneDark адаптирована к Carbon (фон `--bg`, не дефолтный).
- **Свернуть:** код >24 строк по умолчанию **свёрнут** до 12 строк + градиент-fade низа… **нет** (градиент запрещён) → вместо fade — hairline + кнопка «Показать ещё N строк». ПРЕДЛОЖЕНИЕ: порог сворачивания 20 строк.
- **Copy:** копирует чистый код (без номеров строк), `Copy`→`Check` на 1.5s.
- **«Открыть в Code»** (`onOpenInPreview` уже в пропсах) — кнопка `ArrowUpRight` «Открыть в воркспейсе Код» → хэндофф (§7).

## 7. Хэндофф чат → воркспейс  ⭐

В мокапе AI пишет: «Запускаю в воркспейс `Documents`. Готова через 2 минуты.» Это **межворкспейсный хэндофф** — ключевой паттерн продукта.

**7.1 Как выглядит в ленте.**
Внутри AI-сообщения, где модель решает делегировать, после текста — **карточка-артефакт** (не просто inline-`<code>`):
```
┌─────────────────────────────────────────────┐
│ [FileText cyan→indigo]  Investor Deck.pptx   │  ← ico воркспейса Documents
│ ▸ ВОРКСПЕЙС ДОКУМЕНТЫ · СОЗДАЁТСЯ            │  ← mono, цвет --on-bg-dim
│ ▰▰▰▰▰▱▱▱  ~2 мин                            │  ← hairline-прогресс 2px (НЕ спиннер)
│                          [Открыть →]         │  ← ArrowUpRight, ведёт в /documents
└─────────────────────────────────────────────┘
```
- Карточка: `--bg`, бордер `1px --line-bg`, radius 10px, `padding 12px 14px`.
- Иконка артефакта = иконка целевого воркспейса в его цвете (Documents — indigo `#4F46E5`? нет — Документы это инструмент; для генерации PPTX используем ws-corp indigo либо нейтраль; **уточнить** — см. §11). Для Images — `Image` pink, Code — `Code` green.
- **Статусы:** `СОЗДАЁТСЯ` (прогресс-полоса, медленно ползёт по реальному прогрессу, не фейк) → `ГОТОВО` (галочка `Check` `--xero`, кнопка «Открыть →» активна) → `ОШИБКА` (`--sylar`, «Повторить»).
- Прогресс — **hairline-полоса** 2px, заливка `--xero`, фон `--line-bg`. Никаких крутилок.

**7.2 Связь артефакта с диалогом.**
- Артефакт хранит `source_thread_id` + `source_message_id`. В воркспейсе-получателе (A06/A04/A05) у документа — бейдж «▸ ИЗ ЧАТА · Презентация инвесторам» со ссылкой назад в `/chat?thread=…&msg=…`.
- Обратно: в `chat-side` блок «Контекст» получает запись-чип созданного артефакта (`Investor Deck.pptx · ОТКРЫТЬ`), чтобы из диалога был быстрый доступ.

**7.3 Переход.**
Клик «Открыть →» — навигация в целевой воркспейс (`/documents` и т.п.), артефакт уже выбран/открыт. Переход — мгновенная смена роута (не модал). app-side подсвечивает соответствующий воркспейс. **Контекст переносится** (заявлено в IA): целевой воркспейс знает исходный промпт.

**7.4 Авто-роутинг (связь с A12 Empty/Onboarding).**
Если пользователь в `/chat` формулирует задачу, которая явно «не чат» (нарисуй баннер / собери DAG), AI может **предложить** хэндофф до генерации: inline-подсказка «Похоже, это задача для воркспейса Картинки. Перенести? [Да] [Остаться в чате]». ПРЕДЛОЖЕНИЕ — мягкое, не принудительное.

| **8.2 Новый диалог (выбрана модель, 0 сообщений)** | как 8.1, но если открыт существующий пустой thread — заголовок head = «Новый чат», meta = `[МОДЕЛЬ] · 0 ТОКЕНОВ`. Фокус в инпуте. |
| **8.3 Стрим (генерация)** | typing-точки → инкрементальный текст + каретка `▍`; `.send`→`Square`; model-picker disabled; pill «↓ Новое» если юзер проскроллил. (§6.6) |
| **8.4 Стоп пользователем** | частичный ответ + `▸ ОСТАНОВЛЕНО` + «Продолжить»/«Перегенерировать». |
| **8.5 Ошибка модели + фолбэк** | AI-пузырь заменяется на error-стейт: `AlertTriangle` `--sylar` + «Claude Opus 4.7 недоступна (rate limit/нет ключа)». Кнопки: «Повторить» · «Сменить модель ▸» (открывает picker) · «Фолбэк на Sonnet 4.6» (если настроен авто-фолбэк). При авто-фолбэке — тонкая mono-строка в ответе: `▸ ФОЛБЭК · OPUS 4.7 → SONNET 4.6`. |
| **8.6 Лимит токенов / бюджета** | Не-BYOK исчерпал дневной лимит → баннер над инпутом (`--xero-mute` фон, hairline): «Лимит токенов на сегодня исчерпан. Подключите свой ключ (BYOK = ∞) или обновите тариф.» + CTA `→ Настройки`/`→ Тарифы`. Инпут disabled. Окно-контекст переполнен → §6.5 п.2 (>100%). |
| **8.7 Длинный код** | свёрнутый блок 20 строк + «Показать ещё» + Copy + «Открыть в Код» (§6.7). |
| **8.8 Загрузка диалога** | при переключении thread — `MessageSkeleton`×3 (уже есть в `ChatAreaV2`), не спиннер. ≤200ms — без скелета. |
| **8.9 Оффлайн / сеть** | toast-баннер «Нет соединения. Сообщение сохранено как черновик.» Инпут сохраняет текст. |

## 9. Edge cases

| Случай | Решение |
|---|---|
| **Очень длинный диалог (200+ сообщений)** | Виртуализация ленты (react-window / intersection). «Прыжок наверх» (`ArrowUp` floating pill). Свёрнутые ранние сообщения по дате-разделителям. Token-счётчик может превысить окно → авто-усечение старых + плашка «▸ Контекст усечён до окна модели». |
| **50+ диалогов в `chat-list`** | Группировка по датам (§4.1.4) + sticky-заголовки + поиск как основной способ навигации. Lazy-load по скроллу (пагинация). Pin-избранных наверх (ПРЕДЛОЖЕНИЕ: `Pin` в hover-меню `li`). |
| **Обрыв стрима (сеть/таймаут)** | Сохранить частичный текст + `▸ ОБРЫВ СОЕДИНЕНИЯ` (`--sylar`) + «Продолжить». Не терять введённое. Авто-ретрай 1 раз (тихо). |
| **Недоступная модель** | В picker — приглушена + «Нужен ключ»/«Pro». Если активная модель стала недоступна между сообщениями — на отправке предложить фолбэк (§8.5), не блокировать наглухо. |
| **Пустой поиск (нет совпадений)** | В `chat-list` — `--on-bg-dim` mono: «Ничего не найдено по „запрос“». Кнопка «Сбросить». При этом — ПРЕДЛОЖЕНИЕ: предложить «Искать по содержимому сообщений» (полнотекст), а не только по заголовкам/превью. |
| **Очень длинное название/превью** | `ellipsis` (уже в CSS `.nm`/`.pre`), полное — в `title`/тултипе. |
| **Сообщение с гигантской таблицей/кодом** | горизонтальный скролл внутри блока, не растягивает колонку; max-width пузыря 80% соблюдается. |
| **Вложение превышает размер** | чип с ошибкой (`--sylar`) «Файл >20МБ» + удалить. |
| **Параллельные стримы (несколько вкладок)** | блокировка отправки в том же thread из второй вкладки (lock) либо merge — ПРЕДЛОЖЕНИЕ: показывать «Диалог открыт в другой вкладке». |
| **RTL/очень узкий контент-фрейм <360px тела** | пузыри на 100% ширины, аватары над телом. |

## 10. Адаптив

### ≥1280px — полная 3-колоночная раскладка
`app-side 220px` (или icon-rail 56px на 1280–1439, ПРЕДЛ.) + `chat-list 280` + `chat-area 1fr` + `chat-side 280`. Всё как в мокапе.

### 768–1279px (планшет / узкий ноут)
- **`chat-side` сворачивается** в кнопку. Триггер в `chat-head`: справа — chip-кнопка `[МОДЕЛЬ ▾]` (показывает активную модель) + иконка `SlidersHorizontal`. Клик → right-rail выезжает оверлеем поверх ленты (ширина 300px, `--bg-soft`, hairline-лево), затемнение справа нет (без glass). Содержит «Модель» + «Контекст».
- `chat-list` остаётся (или сворачивается в 64px icon-rail аватаров диалогов на 768–1023, ПРЕДЛ.).
- `app-side` → icon-rail 56px.

### <768px (мобайл) ⭐
Колонки складываются в **один столбец** = `chat-area` на весь экран. Навигация — нижний таб-бар + bottom-sheets.

- **Базовый экран:** только переписка (`chat-area`) + `chat-input` закреплён внизу (`mobile-safe-bottom` уже в `ChatAreaV2`).
- **Верхняя панель (sticky):** слева `[☰ Menu]` (`Menu`, открывает `chat-list` как **left drawer** — список диалогов + поиск + «Новый чат»). По центру — название диалога (ellipsis). Справа — **chip активной модели** `[Opus 4.7 ▾]` (`--mono`), тап → **bottom-sheet model-picker**.
- **`chat-list` → left drawer (slide-in 280px, 240ms).** Поиск сверху, список, «+ Новый чат» внизу. Закрытие — свайп влево / тап по затемнению.
- **`chat-side` → разбит на bottom-sheets:**
  - **Модель:** bottom-sheet (drag-handle сверху, выезжает на ~60% высоты). Список как ⌘K-палитра: поиск + группы Premium/Standard/РФ/Free + закреплённые. Тап по модели — выбрать + закрыть.
  - **Контекст/вложения:** bottom-sheet с табами Файл/Knowledge/URL/Диалог (§6.3). Триггер — `Plus` в инпуте.
- **Branching на мобайле:** степпер `‹ 2/3 ›` под сообщением (тач-таргеты ≥44px). Карта дерева (опция B) — отдельный full-sheet по `GitBranch`.
- **Citations:** `.src` тап → раскрытие аккордеоном (полная ширина).
- **Хэндофф-карточка:** на всю ширину, «Открыть →» крупной кнопкой.
- **Token-счётчик:** компактно под инпутом (`1.2K / 200K`), цвет-порог как §6.5.
- **Тач-таргеты ≥44px**, инпут не перекрывается клавиатурой (safe-area), отправка — кнопка `Send` всегда видна.

> Брейкпоинты соответствуют Tailwind (`sm 640 / md 768 / lg 1024 / xl 1280`). `chat-side` прячется уже на `< xl (1280)`, `chat-list` — на `< md (768)`.

## 11. Данные

**Thread (диалог):**
```
id · organization_id · title · model_id (текущая) ·
created_at · updated_at · last_message_preview ·
total_tokens · active_path[] (id выбранных сообщений = активная ветка) ·

```
**Message (сообщение):** (близко к `MessageData` в `ChatMessageV2`)
```
id · thread_id · parent_message_id (для branching) ·
sender_type: user|agent|system · sender_name · content (markdown) ·
model · tokens_used · cost_usd · created_at ·
streaming(bool) · stopped(bool) · rated: 1|-1|null ·
sources[] (citations) · artifacts[] (хэндофф) · siblings_count · sibling_index
```
**Source (citation):**
```
id · type: kb|file|url|template|system · title · snippet · ref (url|kb_doc_id|file_id) · index
```
**Model (для picker):**
```
id · name · provider · tier: premium|standard|rf|free ·
context_window (32k|128k|200k|256k|1M) · available(bool, по BYOK/тарифу) ·
needs_key(bool) · gated_by_plan(bool) · cost_tier(1..3) · pinned(bool) · last_used_at
```
**Attachment (контекст):**
```
id · type: file|kb|url|thread · name · size · source_label (коннектор) · status: ready|loading|error · ref
```
**Token / budget:**
```
thread_tokens · model_context_window · draft_tokens (live) ·
daily_used · daily_limit (null при BYOK=∞) · byok(bool)
```
**Artifact (хэндофф):**
```
id · workspace: documents|images|code|video|sound · name · kind (pptx|png|py…) ·
status: creating|ready|error · progress(0..1) · source_thread_id · source_message_id · target_url
```

## 13. Открытые вопросы / ПРЕДЛОЖЕНИЯ

1. **ПРЕДЛОЖЕНИЕ (branching).** Основной UI веток — inline-степпер `‹ n/m ›` под сообщением (вариант A); опциональная карта-дерево `BranchMap` по кнопке `GitBranch` в `chat-head` (вариант B), на мобиле — full-sheet. Regenerate **всегда** создаёт сиблинг (новую ветку), ничего не перезатирая; «Другой моделью»/«Изменить тон» — пункты дропдауна regenerate. *Подтвердить: нужна ли карта-дерево в MVP или достаточно степпера?*

2. **ПРЕДЛОЖЕНИЕ (model-picker, 14 шт).** Right-rail показывает 3–4 закреплённых/недавних + кнопка «Все модели · 14»; полный выбор — командер **⌘K** (`ModelCommandPalette`) с аккордеонами Premium/Standard/РФ/Free, поиском, закреплёнными, фильтром «Доступны мне» и метками `Нужен ключ`/`Pro`. *Подтвердить точный список 14 и их tier/окна по `providers_status.md`; нужен ли цена-индикатор `···`.*

3. **ПРЕДЛОЖЕНИЕ (мобайл).** 3 колонки → один столбец: `chat-list` = left-drawer (☰), `chat-side` = два bottom-sheet (Модель / Контекст), модель переключается через chip `[Opus 4.7 ▾]` в верхней панели → bottom-sheet-палитра. Тач-таргеты ≥44px, инпут с safe-area. *Подтвердить: drawer или full-screen для списка диалогов на мобиле?*

4. **ПРЕДЛОЖЕНИЕ (attach — единый «+ Контекст»).** Свести 4 источника (Файл / Документ из Knowledge / URL / Прошлый диалог) под одну кнопку `+ Контекст` с табами вместо отдельных точек входа; прикреплённое дублируется чипами в блоке «Контекст» right-rail, чтобы был постоянный «что в контексте». *Подтвердить состав источников для MVP (URL и «прошлый диалог» — обязательны?).*

5. **ПРЕДЛОЖЕНИЕ (хэндофф чат→воркспейс).** Делегирование рендерится **карточкой-артефактом** в ленте (ico воркспейса + статус `СОЗДАЁТСЯ/ГОТОВО/ОШИБКА` + hairline-прогресс 2px + «Открыть →»), артефакт двусторонне связан с диалогом (`source_message_id` ↔ бейдж «▸ ИЗ ЧАТА» в воркспейсе) и появляется чипом в блоке «Контекст». *Открытый вопрос: какой воркспейс/цвет у генерации PPTX — «Документы» (инструмент, нейтраль) или CORP indigo? И прогресс реальный или индикативный?*

6. **ПРЕДЛОЖЕНИЕ (citations).** `.src` кликабелен → раскрывается аккордеон со списком источников (иконка типа + заголовок + сниппет + `ExternalLink`); поддержка нескольких (`▸ SOURCE · 3`) и инлайн-маркеров `[1][2]`, скроллящих к источнику. *Подтвердить: показывать сниппет-цитату или только ссылку в MVP.*

7. **ПРЕДЛОЖЕНИЕ (token-счётчик 3 уровня).** Per-thread в `chat-head` (как в мокапе) + per-input оценка против окна модели с порогами `--sylar` на 75/95/100% + (для не-BYOK) дневной бюджет hairline-полосой / `BYOK · ∞` при своих ключах. *Подтвердить: показывать ли per-message стоимость рядовому юзеру или только за тогглом.*

8. **ПРЕДЛОЖЕНИЕ (аватар юзера `.av.me`).** В мокапе фон аватара — `linear-gradient(135deg, ws-text, ws-images)`, что формально спорит с запретом градиентов. Предлагаю заменить на плоский `--steel` (или `--xero-mute`) с инициалом «V». *Решение за владельцем: оставить мини-градиент на 28px-аватаре как исключение или сделать плоским.*

9. **ПРЕДЛОЖЕНИЕ (интеграция токенов).** V2-компоненты живут на токенах `--bg-base/--text-primary/--accent-blue/...`, а дизайн A03 — на v3-токенах `--bg/--on-bg/--xero/...`. Нужен явный мэппинг-слой (v3→V2) либо внедрение v3-переменных в тему перед декомпозицией. *Это блокер реализации — решить до старта верстки.*

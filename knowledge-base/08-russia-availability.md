# Доступность API из России

| Провайдер | API из РФ | Статус |
|-----------|----------|--------|
| **OpenAI** | НЕТ | Россия в явном блоклисте |
| **Anthropic** | НЕТ | Россия не в 195+ поддерживаемых странах |
| **Google Gemini** | НЕТ | Ограничен в РФ |
| **xAI (Grok)** | НЕТ | Заблокирован из-за санкций |
| **Microsoft Copilot** | НЕТ | Агрессивная блокировка, даже VPN не помогает |
| **Groq** | НЕЯСНО | Нет явной блокировки, но US компания |
| **Stability AI** | НЕЯСНО | Нет явной блокировки |
| **OpenRouter** | ВЕРОЯТНО ДА | Агрегатор, менее строгая проверка |
| **Cerebras** | НЕЯСНО | Новый провайдер |
| **SambaNova** | НЕЯСНО | Новый провайдер |
| **Ollama** | ДА | Локально на компе, нет ограничений |

## Решение для нашей платформы

### Вариант 1: Прокси-сервер (рекомендуемый)
Наш сервер на Yandex Cloud (YOUR_SERVER_IP) выступает прокси:
```
Пользователь (РФ) → наш сервер → API провайдера (US/EU)
```
Пользователь вводит свой API-ключ → мы передаём запрос через сервер → API видит IP сервера, не пользователя.

**Важно:** Может нарушать ToS провайдеров. Но для BYOK модели (ключ пользователя) — серая зона.

### Вариант 2: Сервер в EU/US
Развернуть бэкенд на сервере в Европе (Hetzner, OVH) или US — API будут работать без проблем.

### Вариант 3: Фокус на доступные провайдеры
- **Ollama** — работает всегда (локально)
- **OpenRouter** — вероятно работает
- **Groq/Cerebras/SambaNova** — нужно тестить

## Рекомендация
Перенести бэкенд на европейский сервер (Hetzner ~€5/мес) или использовать текущий сервер как прокси. Для пользователей из РФ это решит проблему доступа ко всем API.

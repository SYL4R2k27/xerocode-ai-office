03 — Common DNA

Что связывает
оба продукта.

Несмотря на разные аудитории, продукты разделяют одно ядро: технологические выборы, дизайн-язык, принципы внедрения. Это и есть SYLAR-DNA.

01

Custom fine-tuned LLMs

Обучаем собственные модели на базе открытых архитектур (Llama, Qwen, GigaChat-base). Специализированные под задачи продуктов. Развёртываются on-prem. Никаких внешних API там, где это критично.

02

5-слойная Privacy Architecture

Pre-egress filter · on-prem engine · tiered access · customer audit log · differential privacy. Любые данные клиента проходят через эти 5 слоёв. Архитектура одна — для XeroCode и RoleFlow.

03

Российский tech-стек

Yandex Cloud · 1С · Битрикс24 · Диадок · СБИС · GigaChat · YandexGPT · КЭП. Реестр российского ПО. ФЗ-152 уровень 1. 44-ФЗ / 223-ФЗ ready. Не «адаптировано после», а спроектировано для РФ изначально.

04

Privacy-by-design аналитика

Все продуктовые метрики на уровне ролей и команд (минимум 5 человек). Никогда не строим «рейтинг сотрудника по AI-производительности». Per-user данные доступны только самому пользователю.

05

Один дизайн-язык

Space Grotesk display · Inter body · JetBrains Mono technical. Carbon + Cream surfaces. Sienna как основной акцент. Браузер-frame для мокапов. Без italic. Без cyberpunk. Tech-studio эстетика.

06

Audit-first onboarding

Любой клиент начинает с измерения, а не с фичи. Express-аудит в обоих продуктах. Сначала — где теряете время / куда утекают данные. Потом — что внедряем для исправления.

04 — Как мы строим

Принципы инженерии
и продукта.

SYLAR — это не «AI стартап с двумя продуктами». Это инженерная студия с зафиксированной методологией. Шесть принципов, по которым мы делаем все решения.

▸ 01 · BORING TECH

Скучные технологии

FastAPI, PostgreSQL, Redis, React, Vite. Никаких экзотических frameworks ради хайпа. Стек, на котором работают тысячи компаний, и где есть найм.

▸ 02 · DESCRIPTIVE

Показываем, не предписываем

Не «вы должны изменить процесс». А «вот что у вас сейчас происходит — решайте». Бренд Bloomberg, не McKinsey. Аудит — измерение, не консалтинг.

▸ 03 · OPEN SOURCE BASE

На открытых моделях

Llama, Qwen, Mistral как фундамент. Свои fine-tunes сверху. Не зависим от ценовой политики OpenAI/Anthropic. Можем гарантировать on-prem.

▸ 04 · RUSSIAN-FIRST

Русский рынок первым

Compliance, оплата, документы, юр. форма — спроектированы под РФ-реалии. Не клон западного продукта с переведёнными формулировками.

▸ 05 · MEASURE OR DON’T BUILD

Измеряем эффект

Каждая фича в продукте — с привязкой к метрике (часы, доллары, риск). Если эффект не измеряется — фичи нет. Это закрывает «AI-вашoo» наглухо.

▸ 06 · SHIP NARROW

Узкий MVP, потом масштаб

RoleFlow стартует с одной индустрии (мебель). XeroCode — с одной аудитории (соло-проф). Только после этого — adaptive engine и расширение.

05 — Путь

Где мы и куда идём.

SYLAR-roadmap покрывает оба продукта. Сначала каждый закрепляется в своей аудитории. Затем — общая платформа: shared identity, billing, custom-LLM hub, partner network.

Q1 2026 · DONE

Brand · research · pre-product

Зонтичный бренд, оба продукта спроектированы, audit-first методология сформирована. XeroCode v3 деплой на xerocode.ru. RoleFlow — pre-product brand foundation.

Q2 2026 · LIVE

XeroCode growth · RoleFlow pilot

XeroCode — рост до 1000 paying seats, 8 workspace в полную силу. RoleFlow — 3 пилотных мебельных клиента по success-fee. Shared infra: auth, billing.

Q3—Q4 2026

Custom LLM hub · vertical packs

Собственный fine-tune layer для обоих продуктов. RoleFlow расширяется на 4 индустрии. XeroCode — telegram-bot для соло-проф.

2027

SYLAR Cloud · partner network

Единая платформа: один SSO для XeroCode + RoleFlow. Партнёрская сеть интеграторов для enterprise. SOC 2 Type II. Реестр российского ПО.

2028+

СНГ → MENA → APAC

Международная экспансия с готовой product-led моделью. Adaptive Engine у RoleFlow работает на любую индустрию без ручного template'а.

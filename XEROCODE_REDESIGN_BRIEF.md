## 1. Позиционирование (одной фразой)

> **XeroCode — один из двух продуктов SYLAR.** Цель: solo-пользователи и SMB-команды до **50 seats**. Задача продукта — *«чтобы один человек мог сделать работу пятерых, не теряя времени и качества»*.

Связка брендов: **SYLAR** (зонт) → **XeroCode** (этот) + **RoleFlow** (второй продукт). Общая база-палитра и типографика у всех трёх; у XeroCode уникальный акцент — Violet.

### Колонка 1 — `chat-list` (история диалогов)
- Поиск по диалогам сверху (`🔍 Поиск по диалогам…`).
- Список: `nm` (название) · `pre` (превью первой строки) · `ts` (время: 14:32 / Вчера / 2 дня).
- Активный диалог подсвечен.

### Колонка 2 — `chat-area` (переписка)
- **`chat-head`**: заголовок диалога + meta (имя модели · счётчик токенов).
- **`chat-messages`**: два типа сообщений —
  - `.msg.me` — юзер, аватар-инициал «V», текст к правому краю;
  - `.msg` (ai) — аватар «AI», структурированный ответ, `<code>` инлайн, и **источник** `.src` («▸ Source · Шаблон … · данные …») — citation прямо в пузыре.
- **`chat-input`**: плейсхолдер «Опишите задачу или прикрепите файл…» + кнопка отправки `→`.

### Колонка 3 — `chat-side` (модель + контекст)
- **▸ Модель**: карточки `.model-pick` — имя + meta (tier-badge `Premium`/`Standard`/`РФ` · окно контекста `200k`/`1M`/`32k`/`Free`). Активная подсвечена. В мокапе: Opus 4.7, GPT-5.4, Sonnet 4.6, Yandex GPT, GigaChat Lite.
- **▸ Контекст**: «+ Документ из Knowledge» — прикрепить RAG-контекст к диалогу.

### Маппинг на существующие V2-компоненты
| Зона мокапа | V2-компонент | Действие |
|---|---|---|
| `chat-list` | `SidebarV2` (или новый ConversationListV2) | стилизовать под 280px bg-soft, поиск, превью+ts |
| `chat-area` шапка+лента | `ChatAreaV2` + `ChatMessageV2` | добавить meta-счётчик токенов, `.src` citation, `me/ai` варианты |
| `chat-input` | `ChatInputV2` | плейсхолдер + attach + send `→` |
| `chat-side` | **НОВЫЙ компонент** | model-picker + context-attach панель (220–280px) |

### Функционал из спеки воркспейса (что должно быть, даже если не нарисовано)
- **14 моделей** через единый чат.
- **История** диалогов (колонка 1).
- **Ветки / branching** — переключение веток ответа. *В статике не показано → требует дизайна (где живёт переключатель ветки?).*
- **Поиск по диалогам** (есть поле, нужна логика/выдача).
- **Перенос контекста** между воркспейсами (заявлено в IA — «контекст переносится»).

### 🔧 Открытые вопросы дизайна чата (на обсуждение)
1. **Branching** — где UI веток? (под сообщением `‹ 2/3 ›`? отдельная мини-карта диалога?)
2. **Right rail на мобиле** — model-picker и контекст: bottom-sheet? скрытая панель?
3. **Model-picker** — 14 моделей не влезут в столбик. Группировка (Premium / Standard / РФ / Free)? поиск? «закреплённые»?
4. **Attach/контекст** — что можно прицепить: файл, документ из Knowledge, URL, прошлый диалог? единый «+ Контекст»?
5. **Citations `.src`** — формат источника, кликабельность, раскрытие.
6. **Token-счётчик** — что показываем: per-message, per-thread, лимит/остаток бюджета?
7. **Состояния** — пустой чат, генерация (стрим), ошибка модели/фолбэк, длинный код-блок.
8. **Перенос в воркспейс** — в мокапе AI пишет «Запускаю в воркспейс Documents» → как выглядит хэндофф чат→документ?

## 10. Связь с реализацией + следующие шаги

**Что есть в коде:** showcase-роут `?view=v3` рендерит весь мастер как есть (namespaced CSS). Прод-чат живёт на V2-компонентах.

**Развилка (решить на обсуждении):**
- **A. Декомпозиция** — разобрать HTML-мокап на настоящие React-компоненты (V2 → V3), токены в CSS-переменные, по одному воркспейсу за раз. Начать с чата.
- **B. Раскатка showcase как прод** — быстрее, но это «псевдо-React» через `dangerouslySetInnerHTML`, не масштабируется на интерактив.

**Рекомендуемый порядок проработки (по словам владельца):**
1. **Чат** — раскладка ✓, добить branching / model-picker / контекст / состояния / мобайл.
2. **Корп-сектор** — спроектировать CRM/Kanban/документооборот/HR с нуля (нет мокапа).
3. **Лендинг** — добить секции и копирайт.

**Жёсткое правило проекта:** деплой на прод `xerocode.ru` — только по явной команде владельца.

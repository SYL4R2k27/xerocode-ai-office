02 — Продукт

Один продукт.
Два слоя.

RoleFlow одновременно проводит ретроспективный аудит работы команды и контролирует все исходящие AI-запросы. Эти два слоя работают друг на друга: аудит подсвечивает узкие места, AI-контур закрывает их безопасно.

01 — Audit Layer

Ретроспективный анализ работы команды.

Изучаем фактическую работу компании за 3–6 месяцев. Строим карту реальных ролей. Считаем потери человеко-часов на микрорутине. Находим скрытые должности внутри сотрудников.

  • Process Loss Audit — ищет где теряется время
  • Hidden Roles Map — кто что реально делает
  • Microbug Catalogue — повторяющиеся мелкие ошибки
  • Time-saved Calculator — измеряет эффект внедрения

02 — Safe AI Layer

Контролируемый проход к любой нейросети.

Перехватываем все исходящие AI-запросы сотрудников. Маскируем PII, секреты, финансовые данные. Маршрутизируем к нужной модели. Проверяем выходы Quality Gate. Логируем каждый шаг.

  • AI-DLP Gateway — маскирование данных перед egress
  • Quality Gate — проверка выходов до клиента
  • Model Router — нужная модель под нужную задачу
  • Audit Trail — неизменяемый журнал на 5 лет

05 — Quality Gate

Микробаги — главная скрытая стоимость AI-внедрения.

На пилоте в юр. фирме (180 seats) Quality Gate поймал ошибки в 87 из 100 проверенных AI-выходов. Выдуманные ссылки на статьи ГК РФ. Двойной НДС в коммерческих предложениях. Несуществующие имена сотрудников в письмах клиентам.

Без Quality Gate эти ошибки уходят наружу. С Quality Gate — ловятся за 200 миллисекунд до отправки.

  • 01

    Hallucination check

    Cross-model review: если основной ответ дал Claude — проверяет GPT. И наоборот. Ищет факты, которых нет в источнике.

  • 02

    Citation check

    Все цифры, ссылки, цитаты сверяются с приложенным контекстом. Выдуманные URL и фабрикованные источники ловятся автоматически.

  • 03

    Tone check

    Embedding-similarity к корпоративным образцам tone-of-voice. Загружается один раз — 50 эталонных писем. Дальше работает на лету.

  • 04

    Factual rules

    Кастомные правила компании: «не обещай скидку больше 15%», «не упоминай конкурентов», «всегда указывай 152-ФЗ оговорку».

07 — Privacy Architecture

Доступ к 3–6 месяцам данных. Без утечек.

Глубокий аудит требует доступа к перепискам, файлам, истории заказов. Это серьёзный privacy-блокер. Мы решаем его архитектурно — пятислойной защитой, через которую данные проходят перед тем как попасть в AI-движок.

Каждый слой можно проверить независимо. Любой ИБ-аудитор пройдёт по архитектуре и убедится: данные не вытекают.

  • 01

    Pre-egress Filter

    ФИО → стабильные псевдонимы. ИНН/СНИЛС/карты → REDACTED. Названия клиентов → [CLIENT-A]. AI видит структуру, не людей.

  • 02

    On-premise Audit Engine

    Локальная LLM (наш custom fine-tune на базе открытых моделей). Никаких внешних API в процессе аудита. Данные не покидают периметр.

  • 03

    Tiered Access Mode

    Три режима доступа AI: Hash-only · Structure-only · Full retention. Выбирает клиент. По умолчанию — Structure.

  • 04

    Customer-visible Audit Log

    Клиент видит каждый запрос AI к данным. Может отказать. Опционально — pre-approval на критичные данные.

  • 05

    Differential Privacy в выводе

    Финальный отчёт без per-person данных. Минимум 5 человек в любой агрегации. GDPR Art. 22 + ФЗ-152 одновременно.

13 — Часто спрашивают

Вопросы. И ответы.

RoleFlow видит содержимое промптов и переписки?

Это решает клиент. Три режима: Hash-only (только метаданные), Structure-only (метаданные + структура процессов без содержимого), Full retention (полный текст в HSM-зоне с подписью клиента). По умолчанию — Structure-only. Любой режим проходит через 5-слойную privacy-архитектуру.

Что включает success-fee пилот?

Старт — бесплатно. Внедрение и инфраструктура — за наш счёт. По истечении 3 месяцев измеряется реальная экономия человеко-часов. Если экономия меньше 100 часов — клиент не платит ничего. Если больше — RoleFlow получает долю по результату по индивидуально согласованной формуле. Доступно для первых 3 пилотных клиентов в каждой новой индустрии.

Как обходим privacy-blocker для 3–6 месячной ретроспективы?

Архитектурно. Перед попаданием в Audit Engine данные проходят через 5 слоёв защиты. Pre-egress filter заменяет ФИО на стабильные псевдонимы, ИНН/СНИЛС/карты — на REDACTED-токены, имена клиентов — на CLIENT-A/B/C. On-prem Audit Engine (наша custom fine-tune модель) работает локально, без внешних API. Differential privacy в выводе гарантирует минимум 5 человек в любой агрегации.

Что за «custom fine-tuned LLM» в Enterprise?

Для on-prem deployment мы тренируем собственную модель на базе открытых архитектур (Llama, Qwen, GigaChat-base). Она специализирована именно на задачах аудита ролей и поиска микробагов — это компактнее и точнее чем универсальный GPT-5/Claude. Развёртывается на железе клиента, никаких внешних API не требует.

Что произойдёт, если RoleFlow Gateway упадёт? AI перестанет работать?

Нет. Gateway работает в режиме fail-open для разрешённых ролей: если control plane недоступен больше 30 секунд, запросы идут напрямую в провайдер с пометкой «degraded mode». Audit-лог пишется локально и синхронизируется при восстановлении. Для критических ролей можно включить fail-closed.

Сколько занимает внедрение?

Express Audit — 7 дней. Role Audit 100 — 4 недели. Полный roll-out Core (после аудита) — 6–8 недель поэтапно по отделам. Полный SOC 2 / ФЗ-152 audit-ready — 12 недель. На enterprise-тарифе ведёт customer success engineer.

Можно купить через тендер / 44-ФЗ?

Подготовка ведётся параллельно с коммерческим запуском. Реестр российского ПО — Q4 2026. Поставка через 44-ФЗ / 223-ФЗ — с Q1 2027. До этого — прямые контракты с частными клиентами. Юр. лицо — ООО «Сайлар Групп», ИНН по запросу.

Positioning.

RoleFlow — это operating layer для AI-работы в крупной компании. Не одна функция, а четыре столпа: контроль доступа, ускорение процессов, проверка качества и измерение эффекта. Все AI-инструменты сотрудников работают под одним контуром.

4 столпа RoleFlow — четыре зачем

Кто, какие модели, какие данные. RBAC + DLP + audit. Shadow AI закрыт.

▸ PILLAR 02

Готовые AI-сценарии под каждую роль: HR-онбординг, финансовое закрытие, sales follow-up.

→ Workflow Studio · Templates · Coach

▸ PILLAR 03

→ Quality Gate · Citation Check · Diff

Кто и зачем покупает RoleFlow

▸ CISO

Видимость + контроль.

Закрывает Shadow AI, готовит SOC 2 / ФЗ-152 за один клик, защищает PII. Pillars: Govern.

▸ COO / HRD

Скорость + качество.

Каждая роль получает AI-плейбуки. Quality Gate ловит ошибки до того как они уйдут наружу. Pillars: Accelerate · Review.

▸ CFO / CIO

ROI + метрика.

Сколько часов сэкономлено, какая инвестиция в AI окупилась, где надо учить, где менять процесс. Pillars: Measure.

Чего RoleFlow НЕ делает

  • ✕ Не заменяет AI-провайдера. Мы не учим foundation-модели. OpenAI, Anthropic, Google, Yandex, GigaChat — продолжают работать как работали.
  • ✕ Не «ещё один SaaS поверх SaaS». Работаем в двух режимах: Overlay (поверх 1С/Битрикс/AD) или Standalone (полная платформа).
  • ✕ Не блокирует AI. Цель — дать видимость, ускорение и контроль качества. Блок — частный случай policy.
  • ✕ Не следит за сотрудниками. Метрики — на уровне ролей и команд, не на одного конкретного человека. Privacy-by-design.

Voice & tone.

RoleFlow говорит как senior security engineer, который объясняет что-то CFO. Точно, без маркетингового шума, без нагнетания страха. Цифры — факты — действия.

▸ КАК ГОВОРИМ

  • Конкретно. «23,4% сотрудников использовали ChatGPT в последний квартал» — не «многие».
  • Технически. Не упрощаем для CISO/CIO. Они понимают слова RBAC, DLP, SAML.
  • Сдержанно. Никаких «революционно», «уникально», «лидер рынка».
  • На English-headlines OK; body — русский для российского рынка.
  • Цифры — всегда с источником: «Gartner 2025», «McKinsey AI Survey».

▸ КАК НЕ ГОВОРИМ

  • Без FUD. Не пугаем «утечкой» в каждом параграфе. Один раз достаточно.
  • Без emoji в копии. Только функциональные иконки в UI.
  • Без «AI-powered», «next-gen», «cutting-edge». Это уже cringe.
  • Без обещаний 100%. Говорим «снижает на 87%», не «полностью устраняет».
  • Без сленга. CISO 50+ читает — слов «сторис», «вайб» быть не должно.

▸ PILLAR 01 — GOVERN · контроль доступа и данных

⌘ 01 / DISCOVERYPASSIVE

Shadow AI discovery.

Пассивный сниффинг DNS / SNI / proxy-логов. Находит chat.openai.com, claude.ai, gemini.google.com и 47+ других AI-доменов. Карта использования по отделам.

  • Без агентов на endpoints
  • 14 дней рекомендуемая discovery-фаза
  • Экспорт: CSV · Splunk · QRadar

⌘ 02 / GATEWAYACTIVE

AI Gateway proxy.

Single endpoint для всех AI-запросов. OpenAI-compatible API. Маршрутизирует к 14 провайдерам через единый ключ. Кэш + rate limiting.

  • OpenAI · Anthropic · Google · Yandex · GigaChat
  • SDK: Python · Node · Go · Java
  • p99 latency overhead < 30ms

⌘ 03 / POLICYDLP

Policy engine.

30+ pre-built правил: PII (ИНН/СНИЛС/паспорт), карты, секреты, финансовая отчётность, исходный код. Custom правила на YARA/Rego.

  • Real-time redaction перед отправкой
  • LLM-классификатор контекстных нарушений
  • Verdicts: allow · redact · flag · block

⌘ 04 / AUDITIMMUTABLE

Audit trail.

Каждый запрос: who · when · what · model · verdict · cost. Hash-chain. Retention 5 лет. SIEM stream + SOC 2 / ФЗ-152 reports one-click.

  • Sigstore-подписи каждой записи
  • S3-WORM bucket · холодный архив
  • SOC 2 / ISO 27001 / ФЗ-152 ready

▸ PILLAR 02 — ACCELERATE · готовые AI-сценарии под каждую роль

⌘ 05 / WORKFLOWDAG

Workflow Studio.

Drag-drop конструктор AI-цепочек: «прочитал письмо → достал номер заказа из 1С → собрал ответ → отправил». 50+ готовых блоков. Без программирования.

  • DAG-оркестрация · до 100 шагов
  • Триггеры: webhook · cron · email · 1C event
  • Версионирование · A/B · rollback

⌘ 06 / TEMPLATESROLE-AWARE

Role playbooks.

120+ готовых сценариев под роли: HR-онбординг, финансовое закрытие месяца, sales follow-up, юридический ревью договора, поддержка клиента L1.

  • Промпты с подстановкой из CRM/ERP
  • Версии для разных индустрий
  • Marketplace · можно публиковать свои

⌘ 07 / COACHUPSKILL

Coach mode.

Сотрудник пишет промпт — Coach подсказывает: «добавь контекст должности», «укажи формат ответа», «не забудь про tone of voice». Учит лучшим промптам.

  • Inline-подсказки в реальном времени
  • Уровни: junior → senior
  • Сертификация · «AI Power User»

⌘ 08 / KNOWLEDGERAG

Corp Knowledge.

Корпоративные документы (1С, Confluence, SharePoint, ЭДО) в одном поиске. RAG с привязкой к роли: HR не видит финансы, FIN не видит личные дела.

  • Connectors: 1C · Bitrix · Confluence · SharePoint
  • Role-aware indexes · per-document ACL
  • Цитирование источника · аудит каждого ответа

▸ PILLAR 03 — REVIEW · ловим ошибки до клиента

⌘ 09 / QUALITYPRE-SEND

Quality Gate.

Перед тем как ответ AI уйдёт клиенту — второй AI-проверяющий ловит микробаги: галлюцинации, неправильные цифры, выдуманные ссылки, неверный tone, фактологию.

  • 4 проверки: hallucination · citation · tone · factual
  • «Diff» между черновиком и финалом
  • Эскалация старшему при low confidence

⌘ 10 / RBACCORE

Role binding.

Каждый сотрудник → роль из AD/SAML → набор разрешённых моделей, шаблонов и данных. HR не видит код, разработчик не лезет в зарплаты.

  • SCIM-синхронизация с AD/Okta/Keycloak
  • 20 pre-built ролей + custom
  • Time-bound elevated access · approvals

▸ PILLAR 04 — MEASURE · считаем эффект, а не «внедрили AI»

⌘ 11 / ANALYTICSROI

Productivity analytics.

Сколько часов сэкономлено каждой ролью, какие задачи AI ускорил больше всего, где не помогает. ROI-дашборд для CFO. Никаких «персональных» метрик — только role-level.

  • Time-saved estimator · по ролям и процессам
  • Heat-map: где AI работает / где простаивает
  • Cost-per-task vs human-baseline

⌘ 12 / CONNECTDEPLOY

Connect mode.

Два режима развёртывания: Overlay (поверх 1С / Битрикс24 / AD без миграции) и Standalone (отдельная платформа, если интеграция не нужна).

  • Overlay: 1С · Битрикс · AD · ЭДО · MS365
  • Standalone: Helm · on-prem · air-gapped
  • Hybrid: миграция за 2-4 спринта

▸ FAQ · ЧАСТО СПРАШИВАЮТ

Вопросы. И ответы.

Quality Gate реально ловит галлюцинации? Как это работает технически? +

Да, четыре проверки на каждый AI-ответ: (1) Hallucination check — второй LLM (cross-model: если основной был Claude, проверяет GPT) ищет факты, которых нет в источнике; (2) Citation check — проверяет что все ссылки и цифры взяты из приложенного контекста, а не выдуманы; (3) Tone check — соответствие корпоративному tone of voice (загружается один раз); (4) Factual rules — кастомные YARA-правила (например «никогда не обещай скидку выше 15%»). При confidence < 0.85 — эскалация старшему. Точность на pilot'ах: 87% поймали реальных багов, 4% false positives.

Productivity analytics — это «следить за сотрудниками»? Как с GDPR / privacy? +

Нет, это не per-employee монитор. Все метрики работают на уровне ролей и команд: «отдел финансов экономит 22ч/неделю», но не «Иван Петров вчера сидел в ChatGPT 3 часа». Privacy-by-design: per-user данные доступны только сотруднику и его супер-админу для своей же роли (можно посмотреть «как я использую»). Соответствует GDPR Article 22 (no automated decision making about individuals) и ФЗ-152 (минимизация ПДн).

Workflow Studio — это no-code? Сможет ли HR-менеджер сделать AI-флоу сам? +

Да, drag-drop. 50+ блоков: «прочитать письмо», «вытащить из 1С», «сгенерировать ответ Claude», «отправить через ЭДО». Никакого Python. Сложные ветвления, циклы, await — всё через UI. Бизнес-роли (HR, FIN) делают флоу за час-два после знакомства. Coach подсказывает «следующий шаг». Для разработчиков — есть YAML-экспорт и CLI для git-flow.

Что такое Overlay режим? Можно объяснить на пальцах? +

Представьте: у вас уже работает 1С, Битрикс24, AD. Вы НЕ хотите ничего переезжать. RoleFlow становится слоем над всем этим: подключается к ним по API, читает события (сделка создана, новый сотрудник добавлен), применяет AI-сценарии и пишет результат обратно. Сотрудники продолжают работать в Битрикс — просто теперь там есть AI-ассистент с правильной ролью, аудитом и Quality Gate. Запуск — 2-3 спринта, без миграций.

RoleFlow видит содержимое промпта? Это конфиденциальные данные. +

Да, проверка содержимого — основа DLP и Quality Gate. Три режима: Hash-only (только метаданные); Encrypted-classify (содержимое расшифровывается только в HSM-зоне, удаляется сразу после classification); Full retention (для регулируемых индустрий, 5 лет в S3-WORM). По умолчанию: Encrypted-classify.

Что произойдёт, если RoleFlow упадёт? AI перестанет работать у всех? +

Нет. Gateway работает в режиме fail-open для разрешённых ролей: если control plane недоступен > 30 секунд, запросы идут напрямую в провайдер с пометкой «degraded mode». Audit-лог пишется локально и синхронизируется при восстановлении. Configurable: для критических ролей (FIN/CRIT) можно включить fail-closed.

Это on-premise или SaaS? Мы в банке — данные не должны покидать РФ. +

Оба. SaaS-инстанс — Yandex Cloud (Москва, ФЗ-152 уровень 1). On-prem — Helm-chart для Kubernetes 1.28+, собранный для air-gapped установок. Минимум: 4 vCPU · 8GB RAM · PostgreSQL 14 · S3-compatible storage. Air-gapped поддерживается полностью — внешние зависимости только AI-провайдеры (вы сами выбираете).

Сколько занимает внедрение? У нас 5000 сотрудников. +

Поэтапно. 14 дней — Discovery (Shadow AI map). 3-4 недели — Gateway active по отделам + первые playbooks для 2-3 ролей. 6-8 недель — полный roll-out, первые ROI-цифры. 12 недель — SOC 2 audit-ready. У вас будет customer success engineer на enterprise-тарифе, который ведёт проект.

Как RoleFlow относится к XeroCode? Это разные продукты или один? +

Разные продукты под одним зонтом SYLAR. XeroCode — AI-офис для команд 1-50 человек: чат, документы, креатив. RoleFlow — operating layer для 50+. RoleFlow Standalone включает XeroCode workspace для пользователей. RoleFlow Overlay работает поверх любого стека, не требует XeroCode. Билинг разный, договоры разные.

Можно купить через тендер / 44-ФЗ? +

Да. RoleFlow зарегистрирован в Едином реестре российского ПО (РЕЕСТР № 22847). Поставка через 44-ФЗ / 223-ФЗ. Оплата по счёту, КС-2/КС-3, акт + документы с НДС. Юр. лицо: ООО «Сайлар Групп», ИНН по запросу.
